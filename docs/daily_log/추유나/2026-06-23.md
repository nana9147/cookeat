    - 해당 type 첫 등록 시 입력값과 무관하게 자동으로 기본 주소 처리

    - 상세주소 저장 시 양끝 공백만 trim, 중간 공백은 보존 (비교 시에만 전체 공백 제거 정규화)
  - `AddressList.tsx` mock 데이터(`MOCK_ADDRESSES`) 제거, `useEffect` cancelled 패턴으로 GET 연동, 로딩 상태 추가
  - **버그 수정**: 등록 후 모달을 다시 열면 이전 입력값이 남아있던 문제 — `AddressForm`의 초기화 `useEffect`가 `address` 변화에만 의존해 등록 모드(`address`가 항상 `undefined`)에서 리렌더되지 않던 것이 원인, `isOpen`을 의존성에 추가해 모달 열릴 때마다 리셋되도록 수정
  - **버그 수정**: 현재 활성 탭(출고지/반품지)과 무관하게 등록 폼이 항상 '출고지'로 열리던 문제 — `Tabs`를 제어 컴포넌트로 전환하고 `AddressForm`에 `defaultType` prop 추가하여 활성 탭 기준으로 기본값 적용

### 4. 주소 수정 API 연동

- **작업**: `app/api/seller/addresses/[addressId]/route.ts`에 PATCH 추가, `db.ts` 추가, `AddressForm.tsx`/`AddressList.tsx` 연동
