- **작업**: `components/ui/Pagination.tsx`
- **내용**:
  - 숫자 버튼(`key={page}`)과 `...`(`key={i}`)의 key가 배열 인덱스 조합에 따라 우연히 겹치는 문제 발견 (currentPage=5, totalPages=10 근처에서 재현)

  - 공용 컴포넌트라 주문/배송/상품/쇼핑 등 페이지네이션을 쓰는 모든 화면에 일괄 적용됨

### 2. 주문 목록 정렬이 페이지 단위로만 적용되던 버그 수정

- **작업**: `supabase/migrations/20260701000000_get_seller_orders_rpc.sql` 신규, `app/api/seller/orders/db.ts`
- **내용**:
  - `order_items` 기준 쿼리에서 조인된 `orders.created_at`(referencedTable)으로 정렬 시, PostgREST의 embedded order + range 조합 한계로 전체 결과 기준이 아닌 페이지 윈도우 내 재정렬만 되던 문제 발견
  - `get_seller_orders` Postgres RPC 함수 신규 작성 — `order_items` + `orders` 조인 후 SQL 레벨에서 정렬 + LIMIT/OFFSET 처리
  - enum(`order_status`)/varchar 컬럼 타입 캐스팅 이슈 다수 수정(`o.status::text`, `p.name::text` 등)
  - `getSellerOrders`(db.ts)를 RPC 호출 방식으로 전환, `SellerOrderRpcRow` 타입 추가

### 3. 정산/대시보드 성능 개선 (코드 리뷰 반영)

  - 버튼 위치: "엑셀로 등록"/"양식 다운로드"는 상품 등록 액션이라 헤더의 "상품 등록" 버튼 옆에, "엑셀 다운로드"는 조회 액션이라 "전체 상품 수" 표시 줄에 배치
