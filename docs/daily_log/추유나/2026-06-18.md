상품 관리 목록 페이지 API 연동 완료 (조회/검색/상태·카테고리 필터/페이지네이션), 카테고리 조회 API 신규 구현, seller/admin 공통 인증 헬퍼 분리

## 세부 작업

### 1. seller/admin 공통 인증 헬퍼 추가

- **작업**: `lib/sellerContext.ts` 신규 구현
- **내용**:
  - `requireSeller`(role 체크)와 별개로, sellerId 확보 로직을 `requireSellerContext`로 분리
  - seller는 `sellers` 테이블에서 `user_id` 기준 조회, admin은 쿼리파라미터 `sellerId`로 지정한 판매자 데이터 조회 가능하도록 분기
  - `serverAuth.ts`는 인증 전용으로 그대로 유지, 도메인 로직(sellerId 조회)만 새 파일로 분리

### 2. 상품 목록 조회 API 구현

- **작업**: `app/api/seller/products/route.ts`, `db.ts` 신규 구현
- **내용**:
  - `requireSellerContext` 가드 적용
  - `keyword`(상품명 검색)/`status`/`categoryId`/`parentId`/`page`/`limit` 쿼리파라미터 처리 (categoryId·parentId는 팀 API 문서에 없던 파라미터로, 카테고리 필터링 위해 추가 정의)
  - `linkedRecipeCount`는 별도 컬럼 없이 `recipe_ingredients`에서 `product_id` 기준 distinct `recipe_id` 카운트로 계산
  - 응답에 `pagination`(`page`/`limit`/`total`/`hasNext`) 포함

### 3. 카테고리 조회 API 구현

- **작업**: `app/api/categories/route.ts`, `db.ts` 신규 구현
- **내용**:
  - 인증 없는 공개 API로 구현 (민감 데이터 아님)
  - DB 구조 확인 과정에서 대카테고리는 `ingredients` 테이블, 소카테고리는 `categories` 테이블(`parent_id`로 `ingredients.ingredient_id` 참조)로 분리되어 있는 구조임을 파악
  - 대카테고리 + 하위 소카테고리 배열을 담은 트리 형태로 응답 구성

### 4. Product 타입 및 ProductStatus 정리

- **작업**: `types/seller/product.ts` 재정의
- **내용**:
  - 실제 `products` 테이블 기준으로 `id`(string) → `productId`(number) 변경
  - `stock`/`status`/`brand` nullable 반영, `image` 필수 처리
  - 카테고리를 `categoryId` + 조인 결과 `categories: { name, parent_id } | null`로 표현
  - `ProductStatus`를 DB enum(`판매중`/`품절`/`판매종료`/`숨김`) 기준으로 재정의 (`판매대기` 제거)
  - `StatusBadge.tsx` 색상 매핑도 위 변경에 맞춰 정리

### 5. getPageNumbers 공통화

- **작업**: `lib/utils.ts`, `hooks/usePagination.ts` 리팩터링
- **내용**:
  - 클라이언트 페이지네이션(`usePagination`)과 서버 페이지네이션(상품 목록) 양쪽에서 쓰이던 페이지 번호 계산 로직을 `lib/utils.ts`의 순수 함수로 분리

### 6. 상품 목록 페이지 API 연동

- **작업**: `app/seller/products/page.tsx`, `ProductTable.tsx` 리팩터링
- **내용**:
  - mock 데이터 제거, 검색/상태/대·소카테고리 필터를 서버 쿼리파라미터로 전달하는 방식으로 전환
  - 실시간 검색 자동완성 제거, Enter 입력 시에만 검색 실행하도록 단순화
  - 대카테고리 선택 시 하위 소카테고리 탭이 나타나는 2단 필터 UI 구현
  - `ProductTable` 내부 클라이언트 페이지네이션 제거 (서버 페이지네이션으로 일원화), `<table>`을 shadcn `Table` 컴포넌트로 교체
  - 로딩 상태 처리 추가

### 7. 테스트 데이터 시딩

- **작업**: SQL Editor로 `ingredients`/`categories`/`products` 테스트 row 삽입
- **내용**:
  - `ingredients`에 대카테고리 7개, `categories`에 소카테고리 다수 row 삽입 (`parent_id`로 연결)
  - 목록/검색/필터/페이지네이션 동작 확인용 `products` 테스트 데이터 삽입

## 특이사항

- `docs/api.md`에 정의된 `/seller/products` 쿼리파라미터(`keyword`/`status`/`page`/`limit`)에 `categoryId`/`parentId`가 추가로 필요해 구현에는 반영했으나, 문서 업데이트는 아직 안 함 — 다음 작업 때 반영 필요
- 상품 등록/수정(POST/PATCH) API는 아직 미착수. `product_images`에서 첫 번째 순서 이미지를 `products.image`(대표이미지)에 동기화하는 로직은 등록/수정 단계에서 처리해야 함
- 상품 삭제(DELETE)는 `ProductTable`에 버튼만 있고 실제 API 연동은 안 된 상태 (콘솔 로그만 찍힘)
