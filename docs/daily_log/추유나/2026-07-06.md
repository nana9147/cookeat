## 세부 작업

### 1. 반응형 브레이크포인트 적용

- app/seller 전반에 max-desktop/max-tablet/max-mobile 브레이크포인트를 적용해 1440px 미만에서도 레이아웃이 깨지지 않도록 수정
- 테이블은 가로 스크롤로, 고정폭 모달은 유연한 너비로 대응

### 2. 모바일 햄버거 메뉴 / 드로어 내비게이션

- 1440px 미만에서는 기존 고정 사이드바 대신 햄버거 버튼 클릭 시 열리는 슬라이드인 드로어(`SellerMobileDrawer`)로 전환
- `SellerMobileNav`를 `SellerMobileDrawer`로 교체(`HeaderMenuButton` 추가)
- 드로어 내 매장 정보 클릭 시 쇼핑몰 이동/로그아웃이 뜨는 동작을 데스크톱 사이드바와 동일하게 맞춤

### 3. 가로 스크롤 시 헤더 잘림 버그 수정

- flex 컨테이너에 min-w-0이 없어 넓은 테이블 콘텐츠가 body까지 확장되며 가로 스크롤 시 헤더가 잘리던 문제 발견
- min-w-0 체인 추가 + body overflow-x-hidden 처리로 수정

### 4. 상품 설명 에디터 기능 추가 및 툴바 UI 개선

- 에디터 내 구분선(Horizontal Rule) 및 형광펜(Highlight) 기능 추가
- 스크롤 시에도 에디터 툴바가 상단에 고정되도록 내부 스크롤 구조 적용

### 5. 상품 등록/수정 API 필수 항목 검증 강화

- 필수 항목 누락 시 어떤 필드가 빠졌는지 한글 레이블로 상세 에러 메시지 반환
- 공백 문자열(스페이스바 입력) 검증을 위한 trim() 로직 추가
- 대표 이미지 유효성 검증 예외 처리 추가

### 6. 재고 수정 시 알림 실시간 미반영 버그 수정

- 알림 드롭다운이 열릴 때(`onOpenChange`)마다 최신 재고 부족 데이터를 API로 재요청하도록 개선
- 수동 새로고침 없이도 재고 수정 후 즉시 알림 목록 및 배지 카운트가 동기화되도록 수정

### 7. 상품 수정 후 페이지 이동 로직 개선

- 수정 후 1페이지로 강제 이동하던 로직 제거
- 현재 보고 있던 페이지 상태를 유지하도록 수정

### 8. Tiptap 패키지 버전 충돌 해결

- @tiptap/core 및 관련 확장 패키지 버전을 3.27.1로 통일
- 빌드 시 발생하던 ERESOLVE peer dependency 에러 수정, npm ci 검증 완료

### 9. 판매자 통계 - 신규/재구매 고객 조회 성능 최적화

- RPC 활용을 통해 신규/재구매 고객 조회 로직을 DB 집계로 전환
- 타입 안정성 강화

### 10. 판매자 대시보드 정산 금액 분리 및 정산일 오프셋 통일

- 대시보드에서 정산예정 금액에 정산대기 금액이 합산되어 노출되던 문제 수정 (`settlementSummary.upcomingTotal` → `scheduledTotal`/`pendingTotal` 분리)
- 대시보드 진입 시에도 `ensureSettlements` 호출하여 정산 상태 자동 갱신
- 정산예정/정산대기 금액을 대시보드 카드에 분리 표시

### 11. 판매자 대시보드 주문 현황 그래프에 판매금액 조회 추가

- 주문건수만 보이던 그래프에서 판매금액도 함께 확인할 수 있도록 개선 (order_items 기준으로 일별 매출 집계 추가)
- 지표(주문건수/판매금액)는 탭으로, 기간(7일/30일)은 별도 드롭다운으로 분리해 두 토글이 겹쳐 보이던 UI 문제 해결

### 12. 셀러 문의관리 페이지 추가

- **작업**: `app/api/seller/inquiries/route.ts`, `db.ts`, `[inquiryId]/reply/route.ts`, `stats/route.ts`, `app/seller/inquiries/page.tsx`, `app/seller/components/Inquiry/InquiryTable.tsx`, `InquiryReplyModal.tsx`, `types/seller/inquiry.ts` 신규
- **내용**:
  - 상품문의(`product_id` 연결)와 배송문의·주문문의(`order_item_id` 연결 + `category`)를 하나의 목록에서 유형 필터로 조회하는 문의관리 페이지 구현
  - 상단 상태카드(전체/미답변/답변완료)는 유형 필터와 무관하게 항상 전체 합계로 고정되도록 별도 `stats` API로 분리
  - 목록에서 모달로 바로 답변 등록/수정 가능(POST/PATCH), 본인 상품·주문 문의인지 소유권 검증 후 처리
  - 1:1 문의 첨부 이미지 썸네일 미리보기 + 클릭 시 라이트박스 확대(좌우 이동 포함) 구현
  - 페이지/테이블/모달 전반에 기존 반응형 브레이크포인트(max-desktop/tablet/mobile) 적용
  - `SellerMobileDrawer`/`SellerSidebar`에 문의관리 메뉴 항목 추가
