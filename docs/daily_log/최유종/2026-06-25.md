## 세부 작업

### 1. 관리자 회원 페이지 등급 필터·에러 처리 개선

**파일**: `app/admin/members/page.tsx`

**문제**

- 등급 필터가 `limit: 1000`으로 전체를 받아 클라이언트에서 JS로 필터링하는 방식
- API 응답 `data.users`, `data.pagination.total`에 null 방어 없음
- `useEffect` catch 블록 미존재로 API 실패 시 빈 화면

**수정 내용**

- 등급 필터를 `grade` 쿼리 파라미터로 API에 위임 (`params.set('grade', filterGrade)`)
- `data.users ?? []`, `data.pagination?.total ?? 0` 으로 null 방어
- `catch` 블록 추가 및 `alert`으로 에러 메시지 표시
- 클라이언트 측 `filtered` / `displayTotal` / `displayMembers` 분기 로직 제거

### 4. 관리자 대시보드 API — 카테고리 조인 경로 수정

**파일**: `app/api/admin/dashboard/route.ts`

**문제**

- `products!inner(ingredients!inner(category))` 잘못된 조인 경로 사용
- 응답 타입을 `any`로 처리하던 코드에서 런타임 에러 발생

**수정 내용**

```ts
// 전
.select('quantity, unit_price, products!inner(ingredients!inner(category))')

// 후
.select('quantity, unit_price, products!inner(categories!inner(ingredients!parent_id!inner(category)))')
```

- `DashCatRow` 타입 정의 추가, 옵셔널 체이닝으로 안전하게 카테고리명 추출
