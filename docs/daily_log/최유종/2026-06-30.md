## 세부 작업

### 1. 어드민 검색 페이지 이중 fetch 제거

**파일**: `app/admin/members/page.tsx`, `app/admin/orders/page.tsx`, `app/admin/products/page.tsx`, `app/admin/sellers/page.tsx`, `app/admin/recipes/page.tsx`

**문제**

검색어가 바뀌면 page-reset용 `useEffect`와 fetch용 `useEffect`가 동시에 실행됨. page가 1이 아닐 때 검색어를 입력하면:

1. fetch useEffect → `fetch(search="김치", page=3)` ← 낡은 page로 불필요한 요청
2. page=1로 리렌더 → fetch useEffect 재실행 → `fetch(search="김치", page=1)`

API가 검색어 변경 1회당 2번 호출됨.

**수정 내용**

- `useRef`로 이전 검색어를 추적하는 `prevSearchRef` 추가
- 별도 page-reset `useEffect` 제거, fetch `useEffect` 하나로 통합
- fetch 내부에서 검색어 변화 감지 후 page가 1이 아니면 `setPage(1)` 후 early return
- 다음 effect 실행(page=1)에서 fetch 1회만 수행

```tsx
// 전 — useEffect 2개
useEffect(() => { setPage(1); }, [debouncedSearch]);
useEffect(() => { /* fetch */ }, [debouncedSearch, ...filters, page]);

// 후 — useEffect 1개
useEffect(() => {
  const isNewSearch = prevSearchRef.current !== debouncedSearch;
  prevSearchRef.current = debouncedSearch;

  async function load() {
    if (isNewSearch && page !== 1) { setPage(1); return; }
    // fetch
  }
  load();
}, [debouncedSearch, ...filters, page]);
```

### 2. 문의 페이지 언마운트 후 setState 방지

**파일**: `app/admin/support/inquiry/page.tsx`

**문제**

`fetchInquiries`가 `useEffect`와 `sendReply` 두 곳에서 호출됨. 로컬 변수 `cancelled`로는 두 호출처를 동시에 가드할 수 없어, 컴포넌트 언마운트 후 fetch가 완료되면 `setInquiries`, `setError`, `setLoading`이 그대로 실행됨.

**수정 내용**

- `cancelledRef = useRef(false)` 추가 — ref는 값이 바뀌어도 리렌더 없이 두 호출처가 공유 가능
- `useEffect` cleanup에서 `cancelledRef.current = true` 설정
- `fetchInquiries` 내 모든 setState에 `if (!cancelledRef.current)` 가드 추가

```tsx
const cancelledRef = useRef(false);

const fetchInquiries = async () => {
  try {
    const { data } = await api.get('/admin/inquiries', { params: { limit: 100 } });
    if (!cancelledRef.current) { setInquiries(data.inquiries ?? []); setError(false); }
  } catch (err) {
    if (!cancelledRef.current) { setError(true); }
  } finally {
    if (!cancelledRef.current) setLoading(false);
  }
};

useEffect(() => {
  cancelledRef.current = false;
  fetchInquiries();
  return () => { cancelledRef.current = true; };
}, []);
```
