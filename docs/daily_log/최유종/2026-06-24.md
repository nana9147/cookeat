**파일**: `app/api/admin/analytics/route.ts`

**추가 내용**

- `GET /api/admin/analytics` — 이번달 / 전월 데이터를 기반으로 아래 지표 계산

### 4. 통계 API 버그 수정 — 전환율 오염 및 전체 테이블 스캔

**파일**: `app/api/admin/analytics/route.ts`

**문제 1**

- `totalUsers = totalUsersResult.count ?? 1` 폴백으로, 활성 유저가 0명이거나 count 조회 실패 시 분모가 1이 되어 전환율이 수천%로 표시

**수정 내용**

```ts
// 전
const totalUsers = totalUsersResult.count ?? 1;
const conversionRate = Math.round((mau / totalUsers) * 1000) / 10;

// 후
const totalUsers = totalUsersResult.count ?? 0;
const conversionRate = totalUsers > 0 ? Math.round((mau / totalUsers) * 1000) / 10 : 0;
```

**문제 2**

- 재구매율 계산을 위해 `orders` 전체를 JS로 가져온 뒤 취소/환불 필터링

**수정 내용**

- DB 레벨에서 `.not('status', 'in', '("취소","환불")')` 필터 적용
- `select('user_id')` 로 필요한 컬럼만 조회

### 5. 판매자 승인+정지 동시 요청 버그 수정

**파일**: `app/api/admin/sellers/[sellerId]/approve/route.ts`

**문제**

- `if (suspend)` 블록과 `if (status === 'approved')` 블록이 `else` 없이 독립 실행
- `{ status: "approved", suspend: true }` 요청 시: 먼저 `suspended`로 저장 → 곧바로 `active`로 덮어써짐

**수정 내용**

```ts
// 전
if (status === 'approved') {

// 후
if (status === 'approved' && !suspend) {
```

- `suspend`가 명시된 경우 자동 활성화를 건너뛰어 `suspend` 의도 보존

### 6. FAQ 수정 API 입력값 검증 누락 수정

**파일**: `app/api/admin/faqs/[faqId]/route.ts`

**문제**

- PATCH 핸들러가 `req.json()` 값을 검증 없이 DB에 바로 전달
- 빈 문자열·잘못된 타입·빈 body도 그대로 저장 가능

**수정 내용**

```ts
if (category === '' || title === '' || content === '') {
  return NextResponse.json({ error: '항목에 빈 값을 입력할 수 없습니다.' }, { status: 400 });
}
if (is_public !== undefined && typeof is_public !== 'boolean') {
  return NextResponse.json({ error: 'is_public은 boolean이어야 합니다.' }, { status: 400 });
}
if (
  category === undefined &&
  title === undefined &&
  content === undefined &&
  is_public === undefined
) {
  return NextResponse.json({ error: '수정할 항목이 없습니다.' }, { status: 400 });
}
```
