## 세부 작업

### 1. 일괄 정산 DB·UI 불일치 수정

**파일**: `app/admin/settlements/page.tsx`

**문제**
- `Promise.all` 사용 시 하나라도 실패하면 전체 롤백되는데, UI 상태는 무조건 전부 완료로 갱신 → 실패 건이 있어도 화면이 정상 처리된 것처럼 보임
- `setStats`에서 `pendingCount: 0`, `pendingAmount: 0`으로 일괄 초기화 → 실패 건의 금액도 완료로 집계

**수정 내용**
- `Promise.all` → `Promise.allSettled`로 교체, 성공한 건만 `succeededIds`로 추출
- 실패 건 수가 있으면 알림 표시
- `setSettlements`: `succeededIds`에 포함된 항목만 상태 변경
- `setStats`: 성공한 건수·금액만 pending에서 차감하고 completed에 합산

### 2. 정산 통계 전체 테이블 스캔 개선

**파일**: `app/api/admin/settlements/route.ts`

**문제**
- 통계 계산을 위해 `settlements` 테이블 전체를 `select('status, amount, fee')`로 조회 후 JS에서 필터링 → 데이터가 많아질수록 성능 저하

**수정 내용**
- 단일 풀스캔 1회 → `Promise.all`로 3개 쿼리 병렬 실행
  - 대기 건: `.eq('status', '대기')` + `count: 'exact'`
  - 완료 건: `.eq('status', '완료')` + `count: 'exact'`
  - 수수료 합계: `.select('fee')` (fee만 선택)
- `count` 값 직접 활용으로 JS 필터링 제거

### 7. 회원 관리 페이지 필터 페이지네이션 수정

**파일**: `app/admin/members/page.tsx`

**문제**
- `grade`는 DB 컬럼이 아닌 서버 계산값(`orderCount >= 10 ? 'VIP' : '일반'`)이라 API 필터가 불가능
- 기존: 서버에서 20개(현재 페이지분)만 받아와 클라이언트에서 등급 필터 적용 → 필터 후 데이터가 20개보다 훨씬 적은데 페이지 수는 DB 전체 기준으로 계산되어 불일치

**수정 내용**
- `filterGrade !== 'all'` 시 `limit=1000`으로 전체 데이터 조회 후 클라이언트 페이지네이션 적용
- `displayTotal`: 등급 필터 켜짐 → `filtered.length`, 꺼짐 → API `total` 사용
- `totalPages`: `displayTotal` 기준으로 재계산
- `displayMembers`: `filtered.slice((page - 1) * PAGE_SIZE, page * PAGE_SIZE)`로 현재 페이지 20개 추출
- `handleFilterGradeChange` 추가: 등급 필터 변경 시 `setPage(1)` 함께 실행 (페이지 리셋 누락 버그 수정)
- `useEffect` 의존성 배열에 `filterGrade` 추가

### 8. 판매자 관리 페이지 필터 페이지네이션 수정

**파일**: `app/admin/sellers/page.tsx`

**문제**
- `filterRating` 활성 시 이미 `limit=1000`으로 전체 조회하는 로직은 있었으나, `filtered.map(...)` 이 조건 맞는 판매자 전부를 한 화면에 렌더링하고 `totalPages`는 API `total`(필터 전) 기준이라 페이지 수 계산 오류

**수정 내용**
- `filteredTotal`: 평점 필터 켜짐 → `filtered.length`, 꺼짐 → API `total` 사용
- `totalPages`: `filteredTotal` 기준으로 재계산 (`Math.ceil(total / PAGE_SIZE)` 제거)
- `pagedSellers`: 평점 필터 켜짐 → `filtered.slice(...)` 클라이언트 페이지네이션 적용, 꺼짐 → 서버 페이지네이션 결과 그대로 사용
- 테이블 렌더링 `filtered.map` → `pagedSellers.map` 교체
- 하단 범위 표시 텍스트 `total` → `filteredTotal`로 교체
