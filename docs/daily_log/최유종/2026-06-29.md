### 3. 리뷰 포인트 적립 race condition 수정

**파일**: `app/api/products/[productId]/reviews/route.ts`, `app/api/recipes/[recipeId]/reviews/route.ts`

**문제**

- `select('point')`로 현재 잔액을 읽은 뒤 `update({ point: 기존 + N })`으로 덮어쓰는 읽고-쓰기(read-then-write) 구조
- 동시 요청이 두 건 들어오면 같은 잔액을 읽어 한 번치 포인트가 사라지는 race condition 발생 가능

**수정 내용**

```ts
// 전 — 읽고-쓰기
const { data: userData } = await supabaseAdmin.from('users').select('point')...
await supabaseAdmin.from('users').update({ point: (userData?.point ?? 0) + pointAmount })...

// 후 — RPC 원자적 증가
await supabaseAdmin.rpc('award_review_point', {
  p_user_id: authed.userId,
  p_amount: pointAmount,
  p_description: '상품 리뷰 작성',
});
```

- 상품 리뷰, 레시피 리뷰 라우트 동일하게 적용

### 4. 리뷰 이미지 URL 검증 추가

**파일**: `app/api/products/[productId]/reviews/route.ts`, `app/api/recipes/[recipeId]/reviews/route.ts`

**문제**

- `images` 배열 원소를 문자열 여부·개수·도메인 검증 없이 그대로 `review_images`에 INSERT
- 임의 문자열·외부 URL이 DB에 저장될 수 있음

**수정 내용**

```ts
const rawImages: unknown[] = Array.isArray(body.images) ? body.images : [];
const storageBase = `${process.env.NEXT_PUBLIC_SUPABASE_URL}/storage/v1/object/public/`;

if (rawImages.length > 5)
  return NextResponse.json({ error: '이미지는 최대 5개까지 첨부할 수 있습니다' }, { status: 400 });
if (!rawImages.every((u) => typeof u === 'string' && u.startsWith(storageBase)))
  return NextResponse.json({ error: '유효하지 않은 이미지 URL입니다' }, { status: 400 });
```
