## 세부 작업

### 1. 쿠폰 지급 방식 전환 — DB 마이그레이션

**파일**: `supabase/migrations/20260701_user_coupons.sql`

**배경**

기존 방식은 사용자가 결제 시 쿠폰 코드를 직접 입력하는 구조였습니다. 사용자 수가 늘어날수록 코드 관리가 어렵고, 동일 쿠폰을 여러 명에게 배포하는 과정이 번거로워 계정 지급 방식으로 전환했습니다.

**마이그레이션 내용**

- Step 1: `user_coupons` 테이블 생성 (`id BIGSERIAL PK`, `user_id`, `coupon_id`, `issued_at`, `used_at`, `UNIQUE(user_id, coupon_id)`)
- Step 2: `orders`에 `user_coupon_id BIGINT FK → user_coupons` 컬럼 추가
- Step 3: `orders.coupon_id` 컬럼 삭제
- Step 4: `coupons.current_usage_count` 컬럼 삭제 (이후 `user_coupons WHERE used_at IS NOT NULL` COUNT로 집계)
- Step 5: `create_order` RPC에 `p_user_coupon_id BIGINT DEFAULT NULL` 파라미터 추가 및 쿠폰 사용 처리 로직 추가

### 2. 어드민 쿠폰 관리 API

**파일**: `app/api/admin/coupons/route.ts`, `app/api/admin/coupons/[couponId]/route.ts`, `app/api/admin/coupons/[couponId]/issue/route.ts`

| 엔드포인트 | 설명 |
| --- | --- |
| `GET /admin/coupons` | 쿠폰 목록 + 지급/사용 통계 조회 |
| `POST /admin/coupons` | 쿠폰 생성 |
| `DELETE /admin/coupons/:couponId` | 쿠폰 삭제 (사용된 쿠폰 삭제 시 409) |
| `POST /admin/coupons/:couponId/issue` | 쿠폰 지급 (`issueAll: true` 또는 `userIds[]`) |

**주요 구현 포인트**

- `issueAll: true` 시 서버에서 `role=user, status=active` 사용자 목록을 직접 조회해 일괄 지급 — 클라이언트에서 userId 목록을 보내지 않아 대규모 사용자에도 안전
- `upsert({ onConflict: 'user_id,coupon_id', ignoreDuplicates: true })` — 중복 지급 방지, 이미 지급된 사용자는 무시
- 삭제 가드: `user_coupons WHERE used_at IS NOT NULL` COUNT로 사용 여부 확인
