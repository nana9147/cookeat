**파일**
- `app/api/admin/faqs/route.ts`
- `app/api/admin/faqs/[faqId]/route.ts`

**추가 내용**
- `GET /api/admin/faqs` — FAQ 목록 조회 (카테고리·키워드 필터)

### 4. `inquiry_replies` 단일 객체 반환 시 `isAnswered` 오작동 수정

**파일**: `app/api/admin/inquiries/route.ts`

**문제**
- Supabase가 1:N 관계에서 결과가 1건일 때 배열 대신 단일 객체를 반환하는 경우 발생
- `Array.isArray(raw)` 체크 없이 `.length`를 참조하면 단일 객체에서는 항상 `isAnswered = false`가 됨

**수정 내용**
- `toInquiry()` 내부에서 `raw`를 `null / 배열 / 단일 객체` 세 케이스로 분기 처리

```ts
const replies: RawReply[] =
  raw === null || raw === undefined
    ? []
    : Array.isArray(raw)
      ? (raw as RawReply[])
      : [raw as RawReply];
```

### 5. 고객센터·FAQ 페이지 에러 핸들링 누락 수정

**파일**: `app/admin/support/page.tsx`, `app/admin/support/inquiry/page.tsx`, `app/admin/support/faq/page.tsx`

**문제**
- fetch 함수에 `finally`만 있고 `catch`가 없어 API 실패 시 사용자 피드백 없음 (빈 목록으로 보임)
- FAQ 추가·수정·삭제 뮤테이션에 에러 처리 없어 실패 시 UI가 멈추거나 잘못된 상태 유지
- `handleAddSave`에서 `await`와 `.then()` 혼용

**수정 내용**
- `support`, `inquiry` 페이지에 `error` state 추가, `catch` 블록에서 `setError(true)` 처리
- 테이블에 로딩 중 / 에러 / 빈 상태 / 목록 네 가지 분기 명시
- FAQ 추가·수정·삭제를 `try/catch`로 감싸고 실패 시 `alert` 피드백
- `handleAddSave`의 `.then()` 혼용을 `async/await`으로 통일
