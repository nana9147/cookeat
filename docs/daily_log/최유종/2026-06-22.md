## 세부 작업

### 1. 리뷰/신고 관리 및 문의 API 라우트 추가

**파일**
- `app/api/admin/reviews/route.ts`
- `app/api/admin/reviews/[reviewId]/route.ts`
- `app/api/admin/inquiries/route.ts`
- `app/api/admin/inquiries/[inquiryId]/reply/route.ts`

**추가 내용**
- `GET /api/admin/reviews` — 리뷰 목록 페이지네이션·키워드 검색 조회 (상품/레시피 조인)
- `DELETE /api/admin/reviews/[reviewId]` — 리뷰 삭제
- `GET /api/admin/inquiries` — 문의 목록 조회
- `POST /api/admin/inquiries/[inquiryId]/reply` — 문의 답변 등록

---

### 2. 리뷰/신고 관리 페이지 실 DB 연동

**파일**: `app/admin/reviews/page.tsx`

**문제**
- 목록 데이터가 하드코딩 mock으로만 동작

**수정 내용**
- `GET /api/admin/reviews` 호출로 목록·페이지네이션 실 DB 연동
- API 응답 필드를 컴포넌트 `Review` 인터페이스로 매핑

---

### 3. 레시피 리뷰 누락 버그 수정

**파일**: `app/api/admin/reviews/route.ts`

**문제**
- `products(name)`, `recipes(title)` 기본 조인이 inner join으로 동작 → 상품과 연결된 리뷰만 조회되고, 레시피 리뷰(products 없음)는 결과에서 누락

**수정 내용**
- `products(name)` → `products!left(name)`
- `recipes(title)` → `recipes!left(title)`
- left join으로 변경하여 상품·레시피 리뷰 모두 조회

---

### 4. 주문 시 재고 소진 상품 자동 품절 전환

**파일**: `app/api/order/route.ts`

**문제**
- 주문으로 재고가 0이 되어도 상품 상태가 여전히 '판매중'으로 유지 → 추가 주문 유입 가능

**수정 내용**
- 재고 차감 후 `prevStock - quantity === 0`인 상품 ID를 추출
- 해당 상품 중 상태가 '판매중'인 것만 '품절'로 업데이트 (숨김 상태는 유지)

---

### 5. 관리자 리뷰 페이지 API 에러 처리 추가

**파일**: `app/admin/reviews/page.tsx`

**문제**
- `load()` 함수에 `catch` 블록이 없어 API 실패 시 unhandled promise rejection 발생, 사용자 피드백 없음
- `data.pagination.total` 접근 시 `pagination`이 undefined이면 TypeError 발생
- `handleDelete`에 에러 처리가 없어 삭제 실패 시 UI 상태 불일치 가능성

**수정 내용**
- `error` state 추가, `load()` 시작 시 초기화
- `load()`에 `catch` 블록 추가 → 에러 메시지 화면 표시
- `data.pagination?.total ?? 0` optional chaining으로 TypeError 방지
- `handleDelete` try/catch 감싸고 실패 시 `alert`으로 피드백, UI 상태 유지

---

### 6. 미구현 블라인드 처리 기능 제거

**파일**: `app/admin/reviews/page.tsx`

**배경**
- `handleBlind`가 UI 상태만 변경하고 API 호출이 없어 새로고침 시 원래 상태로 돌아오는 미구현 기능

**수정 내용**
- `handleBlind` 함수 제거
- 다이얼로그 내 "블라인드 처리" 버튼 제거
