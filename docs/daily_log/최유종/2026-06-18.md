## 오늘 반영된 주요 작업 요약

관리자 주문 관리·정산 관리 페이지를 목업 데이터에서 실 DB로 전환하고, 상품 관리 페이지의 DB 스키마 변경(`ingredient_id` → `category_id`)을 라우트와 페이지에 반영했습니다. `order_items` RLS 정책의 무한 재귀 버그를 DB 레벨에서 수정하고, 관리자 대시보드의 API 오류 처리를 추가했습니다.

## 세부 작업

### 1. 관리자 주문 관리 페이지 실 DB 연동

- `app/api/admin/orders/route.ts` (신규)
  - `GET /api/admin/orders` — 주문 목록 페이지네이션·검색·상태 필터 API 구현
  - `order_items`, `products` 조인으로 주문 상품명 포함 응답
- `app/admin/orders/page.tsx`
  - 목업 데이터 배열 제거, 실 API 연동
  - 검색어 400ms 디바운스 (`useRef` + `setTimeout` 패턴)
  - 필터·검색 변경 시 `cancelled` 플래그로 경쟁 상태 방지
  - loading · error state 및 빈 상태 UI 추가
  - 공통 `Pagination` 컴포넌트 적용

### 2. 관리자 정산 관리 페이지 실 DB 연동

- `app/api/admin/settlements/route.ts` (신규)
  - `GET /api/admin/settlements` — 정산 목록 페이지네이션·기간·상태 필터 API 구현
  - sellers 조인으로 상호명·계좌 정보 포함, 전체 통계(대기/완료 금액·수수료 합계) 함께 반환
- `app/api/admin/settlements/[settlementId]/route.ts` (신규)
  - `PATCH /api/admin/settlements/:id` — 정산 완료 처리 API
  - 이미 완료된 정산 재처리 시 409 응답
- `app/admin/settlements/page.tsx`
  - 목업 데이터 제거, 실 API 연동
  - 단건 정산 완료(`handleSettle`) · 일괄 완료(`handleBulkSettle`) 추가
  - 금액 한국어 포맷 함수(`formatKoreanAmount`) 추가

### 3. 관리자 상품 관리 DB 스키마 변경 반영

- `app/api/admin/products/route.ts`
  - `ingredients` 조인 → `categories` 조인으로 교체 (`ingredient_id` 컬럼 삭제됨)
  - 카테고리 필터 파라미터 `category`(문자열) → `parentId`(숫자)로 변경
  - 응답 필드 `ingredientName/category` → `categoryId/categoryName/parentId`로 변경
- `app/admin/products/page.tsx`
  - `CATEGORIES` 문자열 배열 → `PARENT_CATEGORIES` (id/label 객체 배열)로 교체
  - `filterCategory` → `filterParentId` state로 변경, API 파라미터 `parentId`로 전달
  - 테이블·다이얼로그의 `p.category` → `p.categoryName`으로 교체

### 4. order_items RLS 무한 재귀 버그 수정 (DB)

- **원인**: `order_items: buyer read own` 정책이 `orders`를 서브쿼리로 조회 → `orders: seller read own product orders` 정책이 다시 `order_items`를 조회 → 무한 루프
- **해결**: `get_order_user_id_bypass()` SECURITY DEFINER 함수를 DB에 생성하여 `order_items` 정책에서 `orders`를 RLS 없이 조회하도록 교체
- 코드 변경 없음, DB 정책만 수정

### 5. 관리자 대시보드 API 오류 처리 추가

- `app/admin/page.tsx`
  - `.then()` 체이닝 → `async/await + try/catch`로 교체
  - `loading` · `error` state 추가 및 JSX에서 로딩·에러 화면 분기 처리
  - 기존: API 실패 시 `s.todayRevenue.toLocaleString()` TypeError 크래시 → 수정 후 에러 메시지 표시
