두 RPC 함수와 인덱스 SQL은 `docs/sql/performance-fixes.sql`에 정리해 뒀습니다. Supabase SQL Editor에서 실행해야 실제 동작합니다.

## 세부 작업

### 1. 상품 검색 키워드 이스케이프

- `escapeLike()` 헬퍼 추가: `%`, `_`, `,`, `(`, `)`, `\` 를 이스케이프
- 기존 `ilike('name', `%${keyword}%`)` → `ilike('name', `%${escaped}%`)` 로 교체
- `.or()` 필터에 사용자 입력을 그대로 넣을 때 PostgREST 구문이 깨지는 문제 방지

### 2. 별점순 정렬 DB 처리로 전환

- 기존: 최대 500개 상품과 리뷰를 JS로 받아 메모리 정렬 후 slice
- 변경: `get_products_by_rating` RPC가 products × sellers × reviews를 GROUP BY 집계 후 ORDER BY avg_rating DESC, LIMIT/OFFSET 처리
- RPC 결과에 `total_count`가 포함되어 있어 별도 count 쿼리 불필요
- `RATING_SORT_MAX` 상수 및 관련 메모리 정렬 코드 제거

### 3. count 전략 최적화

- `count: 'exact'`는 조건에 맞는 전체 행을 한 번 더 스캔하는 비용이 있음
- 1페이지(`page === 1`)에서만 `'exact'` 사용, 이후 페이지는 `'estimated'`로 전환
- 검색 결과 페이지 진입 시에만 정확한 total이 필요하고 이후 페이지에서는 근사값으로 충분

### 4. 주문 생성 트랜잭션화

- 기존: 재고 차감(CAS) → 품절 전환 → orders INSERT → order_items INSERT 가 개별 쿼리로 순차 실행, 실패 시 `restoreStock` + `order delete` 수동 보상
- 변경: `create_order` RPC 단일 호출로 교체
  - 함수 내부에서 재고 차감 → 품절 전환 → orders → order_items 를 하나의 트랜잭션으로 처리
  - 재고 부족 시 `raise exception`으로 전체 롤백
  - 프로세스가 중간에 끊겨도 DB가 자동 롤백하므로 데이터 불일치 없음
- `restoreStock` 함수, `DecrementedItem` 타입 제거

### 5. Supabase SQL 파일 작성

- `docs/sql/performance-fixes.sql` 신규 작성
  - `pg_trgm` 확장 + `idx_products_name_trgm` GIN 인덱스 (ILIKE 풀스캔 개선)
  - 기본 인덱스: `idx_products_status_created`, `idx_products_category`, `idx_reviews_product`
  - `get_products_by_rating` RPC 함수
  - `create_order` RPC 함수
