# 2026-06-25 작업 내역

## 오늘 반영된 주요 작업 요약

마이페이지 컴포넌트 구조를 정리하고 신규 서브 페이지들을 추가했습니다. 기존 `components/(auth)/mypage/` 하위에 혼재하던 컴포넌트들을 각 페이지 `_components/` 폴더로 이동해 경계를 명확히 했습니다. 취소/반품 내역·내 레시피 목록·찜 목록·포인트 내역 페이지를 신규 구현했습니다.

API 레이어에서는 상품 검색 SQL 인젝션 방어(escapeLike), 별점순 정렬의 RPC 전환, 주문 생성 로직의 트랜잭션화를 적용해 안정성을 높였습니다. 인증 흐름에서는 로그인 시 `app_metadata.role` fallback 처리를 추가했습니다.

## 세부 작업

### 1. 마이페이지 컴포넌트 구조 재편

- `components/(auth)/mypage/{orders,profile,seller}` → `app/(main)/mypage/{orders,profile,sellerapply}/_components/` 로 이동
- 각 `page.tsx` 임포트 경로 수정
- `OrderStatusTabs`에서 취소 탭 제거 → 별도 `/mypage/cancel` 페이지로 분리

### 2. 취소/반품 내역 페이지 추가

- `/mypage/cancel` 신규 생성
- 취소·반품 탭으로 분리해 각 status 필터 조회
- 기존 `OrderCard`, `OrderPagination` 컴포넌트 재활용

### 3. 내 레시피 목록 페이지 추가

- `/mypage/recipes` 신규 생성
- `MyRecipeList` 컴포넌트: 본인 작성 레시피 목록 조회 및 렌더링

### 4. 찜 목록 페이지 추가

- `/mypage/wishlist` 신규 생성
- 찜한 상품 목록을 API로 조회해 렌더링

### 5. 포인트 내역 페이지 추가

- `/mypage/points` 신규 생성
- `PointHistory` 컴포넌트: 포인트 적립/사용 내역 조회 및 렌더링

### 6. 상품 검색·정렬 개선 및 주문 생성 트랜잭션화

- `products/route.ts`: `escapeLike()` 추가로 검색어 SQL 인젝션 방어
- 별점순 정렬을 `get_products_by_rating` RPC로 전환
- count 전략: 1페이지 `exact` / 이후 `estimated`로 최적화
- `order/route.ts`: 재고 차감·주문 생성을 `create_order` RPC 단일 호출로 교체, 보상 코드(`restoreStock`) 제거

### 7. 로그인 role fallback 처리

- `fix(auth)`: 로그인 시 `app_metadata.role`이 없을 경우 기본값 fallback 추가
