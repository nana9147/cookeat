## 오후 반영된 주요 작업 요약

레시피 작성/수정 기능을 새로 만들었습니다. 재료는 대분류를 먼저 선택한 뒤 그 카테고리로 필터링된 상품을 검색해 연결하고, 마땅한 상품이 없으면 이름을 직접 입력해 등록할 수 있습니다. 조리순서는 제목·설명·팁을 각각 입력받도록 폼을 손봤습니다.

기존에는 재료(`recipe_ingredients`)가 대분류 ID만 가지고 있어 상품 미연결 재료의 구체적인 이름을 저장할 수 없었고, 조리순서(`recipe_steps`)도 제목/설명/팁을 줄바꿈으로 합쳐 `description` 한 컬럼에 저장하고 있어 화면에 그대로 다시 나눠 보여줄 수 없었습니다(HTML은 일반 텍스트의 `\n`을 무시함). 두 테이블에 각각 `name`, `title`/`tip` 컬럼을 추가해 근본적으로 해결했습니다.

레시피 상세페이지도 함께 손봤습니다. 상품이 연결된 재료인데도 대분류명("채소")만 노출되던 버그, 재료·조리순서 리스트의 React key가 대분류 ID를 재사용해 같은 카테고리 재료가 여러 개면 경고가 뜨던 버그를 고쳤습니다. 조리순서 사진은 새 탭 대신 모달로 확대해서 볼 수 있게 했고, 본인이 작성한 레시피는 상세페이지에서 바로 수정 페이지로 이동할 수 있도록 버튼을 추가했습니다.

마지막으로 오늘 작업이 반영된 레시피 작성/상세/수정 화면을 실제 로그인 세션으로 캡처해 포트폴리오 스크린샷을 갱신했습니다.

## 세부 작업

### 4. 레시피 상세페이지 재료·조리순서 표시 개선

- `recipe_steps`에 `title`, `tip` 컬럼 추가 — 기존엔 작성 폼의 제목/설명/팁을 `\n\n`으로 합쳐 `description`에 저장해, 화면에 줄바꿈도 무시되고 팁도 구분되지 않았음
- `lib/serverRecipes.ts`: 재료 표시 이름을 `상품명 → 직접입력명 → 대분류명` 우선순위로 계산하도록 수정 (기존엔 상품이 연결돼 있어도 무조건 대분류명만 노출)
- `RecipeIngredients`/`RecipeSteps` 리스트의 React key를 대분류 ID(`ingredientId`) 대신 각 행의 고유 PK(`id`)로 변경 — 같은 대분류의 재료가 여러 개일 때 key 중복 경고 및 렌더링 오류 가능성 해결
- 상품 미연결 재료는 체크박스·장바구니 버튼·"직접 구매하기"/"재료 묶음 추가" 버튼을 숨기도록 정리 (구매할 수 없는 재료에 구매 UI가 노출되던 문제)
- 조리순서 이미지를 새 탭(`target="_blank"`) 대신 Dialog 모달로 확대해서 볼 수 있도록 변경

### 5. 레시피 작성페이지 추가 (`/recipes/write`)

- 기본정보·재료·조리순서를 입력받아 `POST /api/recipes`로 등록하는 폼 신규 작성
- 재료: 대분류(`ingredient_id`)를 먼저 선택해야 상품 검색이 활성화되고, `GET /api/products`에 카테고리 필터를 함께 넘겨 해당 대분류 상품만 검색되도록 연동. 검색 결과에 없는 재료는 이름을 직접 입력해 등록 가능 (`recipe_ingredients.name`에 저장)
- 조리순서: 단계별 제목/상세설명/팁/사진을 입력받아 그대로 저장 (기존처럼 문자열로 합치지 않음)
- `GET /api/ingredients`: 재료 대분류 옵션 목록 제공 API 신규 추가
- `POST /api/recipes`: 카테고리·재료 존재 여부 검증, 상품 연결 재료는 상품의 실제 대분류로 `ingredient_id`를 서버에서 재계산, 등록 중 실패 시 업로드된 이미지와 레시피 행을 롤백

### 6. 레시피 수정페이지 추가 (`/recipes/[id]/edit`)

- 작성 폼(`RecipeWriteForm`)을 재사용해 기존 값(재료·조리순서·사진 포함)을 프리필하고 수정할 수 있도록 `mode`/`initialRecipe` 지원 추가
- 본인 레시피가 아니면 상세페이지로 되돌리는 클라이언트 가드(`RecipeEditGuard`) 추가 — 실제 인가는 서버가 담당하므로 UX용 가드
- 상세페이지에 본인 레시피일 때만 보이는 "수정" 버튼(`RecipeOwnerActions`) 추가
- `PATCH /api/recipes/[recipeId]`: 로그인한 유저가 작성자 본인인지 서버에서 재검증 후 필드·재료·조리순서를 전체 교체. 사진은 새로 올린 것만 업로드하고 나머지는 기존 URL을 유지하며, 교체·삭제된 이미지는 스토리지에서 함께 정리
- `db.ts`: 카테고리/재료 검증, 재료-상품 매핑 로직을 `createRecipe`/`updateRecipe`가 공유하도록 분리

### 7. 포트폴리오 스크린샷 갱신

- `recipe-write`, `recipe-edit` 캡처 항목 신규 추가
