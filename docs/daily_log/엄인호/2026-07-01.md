# 2026-07-01 작업 내역

## 오늘 반영된 주요 작업 요약

결제 페이지에 포인트·쿠폰 할인 기능을 추가했습니다. 사용자가 보유 포인트를 직접 입력하거나 전액 사용할 수 있고, 쿠폰 코드를 입력해 할인을 적용할 수 있습니다. 적용된 할인은 결제 금액 요약 섹션에 실시간으로 반영됩니다.

포인트·쿠폰 검증은 주문 생성 API 서버 측에서 이중으로 수행하며, DB 트랜잭션(`create_order` RPC) 내에서 포인트 차감, 포인트 내역 적립, 쿠폰 사용 횟수 증가가 원자적으로 처리됩니다.

아울러 토스페이·카카오페이 결제 완료 처리 시 `orders` 테이블에 `payment_key`(결제 게이트웨이 키)가 존재하지 않아 주문 상태가 '결제완료'로 갱신되지 않던 버그를 수정했습니다. 카카오페이 리다이렉트 URL 검증 로직도 하드코딩된 허용 목록에서 도메인 접미사 방식으로 개선했습니다.

오후에는 쿠폰 UX를 코드 직접 입력 방식에서 목록 선택 모달로 교체하고, 카카오페이 결제 승인 시 `orderId` 유실·쿠폰 할인 유형 오판·포인트 이용내역 컬럼명 오류 등 결제 플로우 전반의 버그를 집중 수정했습니다.

---

## 세부 작업

### 1. 포인트·쿠폰 할인 UI (`DiscountSection.tsx`)

- `GET /api/users/me/points`로 보유 포인트 조회 후 표시
- 포인트 직접 입력 / 전액 사용 / 초기화 버튼
- 쿠폰 코드 입력 → 적용 버튼 → 서버 검증 후 할인 금액 표시
- 쿠폰 적용 상태일 때 코드·할인 내용·제거 버튼 표시
- `PaymentMethod` 위쪽에 배치

### 2. 쿠폰 검증 API (`GET /api/coupons/validate`)

- 쿠폰 코드·주문 금액을 받아 유효성 검증
  - 존재 여부, 만료일, 최대 사용 횟수, 최소 주문 금액
- `%` / 원 할인 유형에 따라 `discountAmount` 계산 후 반환

### 3. 주문 API 포인트·쿠폰 처리 (`POST /api/order`)

- `usePoint`, `couponCode` 파라미터 수신
- 서버에서 포인트 잔액 검증, 쿠폰 재검증(만료·횟수·최소 금액)
- `finalAmount = 상품금액 + 배송비 - 포인트 - 쿠폰할인` (음수 방지 `Math.max(0, ...)`)
- `create_order` RPC에 `p_used_point`, `p_coupon_id` 전달

### 4. DB 마이그레이션 (`create_order` RPC 업데이트)

- `orders.payment_key VARCHAR(255)` 컬럼 추가
- `p_used_point`, `p_coupon_id` 파라미터 추가
- 포인트 사용 시: `users.point` 차감 + `point_history` '사용' 내역 INSERT
- 쿠폰 사용 시: `coupons.current_usage_count` 증가
- 포인트 잔액 부족 시 예외(`P0002`) 발생 → 전체 트랜잭션 롤백

### 5. 버그 수정

- **결제 완료 후 주문 상태 미갱신**: `orders.payment_key` 컬럼 부재로 토스/카카오 approve 라우트의 UPDATE가 전체 실패하던 문제 수정. 마이그레이션으로 컬럼 추가 + 업데이트 에러 체크 추가
- **카카오페이 URL 검증 실패**: 하드코딩된 허용 호스트 목록에 없는 도메인(`mockpay.kakaopay.com` 등) 차단 문제 → `.kakao.com` / `.kakaopay.com` 접미사 검증으로 변경

### 6. 쿠폰 선택 모달 도입 (`CouponModal.tsx`)

- **작업**: `app/api/coupons/route.ts` 신규, `CouponModal.tsx` 신규, `DiscountSection.tsx` 개편
- **내용**:
  - 기존 쿠폰 코드 직접 입력 방식을 사용자 보유 유효 쿠폰 목록을 카드 형태로 보여주는 선택 모달로 교체
  - `GET /api/coupons`: 로그인 유저의 유효 쿠폰 목록 반환(만료·최대 사용 횟수 초과 제외)
  - `validate` 엔드포인트에 중복 사용 방지 로직 추가 — `orders` 테이블에서 해당 유저의 해당 쿠폰 사용 이력 체크
  - 결제완료 페이지에서 사용 포인트 차감 내역이 누락되던 문제 수정(`used_point` SELECT 추가)
  - 결제 요약 섹션의 '포인트 적립 예정' 문구 제거 — 실제 적립 로직이 없는 상태에서 표시되던 오해 소지 제거

### 7. 카카오페이 결제 승인 시 `orderId` 유실 버그 수정

- **작업**: `components/(auth)/cart/complete/usePaymentConfirm.ts`
- **내용**:
  - 카카오페이 리다이렉트 URL에는 `pg_token`만 포함되고 `orderId`는 전달되지 않아, approve API 호출 시 `orderId`가 `null`이 되어 404 오류가 발생하던 버그 발견
  - 결제 시작 시 `sessionStorage`에 `kakaoOrderId`를 저장해두고, 리다이렉트 복귀 후 폴백으로 복원하도록 수정

### 8. 쿠폰 `discount_type` 비교값 불일치 수정

- **작업**: `app/api/coupons/route.ts`, `validate/route.ts`, `order/route.ts`, `CouponModal.tsx`, `DiscountSection.tsx` (5곳)
- **내용**:
  - DB에는 `discount_type`이 `'rate'`로 저장되어 있는데 코드 전반에서 `'%'`로 비교하고 있어 % 할인 쿠폰이 정액 할인처럼 잘못 계산되던 버그 수정
  - 일괄 `'%'` → `'rate'`로 교체

### 9. 포인트 이용내역 조회 컬럼명 오류 수정

- **작업**: `app/api/users/me/points/route.ts`
- **내용**:
  - 포인트 내역 조회 쿼리에서 `select('id')` → `select('point_id')`로 수정
  - `historyRes.error` 미체크로 인해 조회 실패가 조용히 빈 배열로 반환되던 문제 함께 수정

### 10. 결제 API `req.json()` 파싱 오류 처리 추가

- **작업**: `app/api/order/route.ts`, `payment/kakao/approve/route.ts`, `payment/kakao/ready/route.ts`, `payment/toss/confirm/route.ts`
- **내용**:
  - `req.json()` 호출 시 Content-Type 불일치나 빈 바디로 파싱이 실패할 경우 처리되지 않고 500으로 떨어지던 문제 수정
  - 4개 결제 관련 라우트에 `try/catch` 추가, 파싱 실패는 400으로 응답
