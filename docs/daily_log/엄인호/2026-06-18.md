장바구니·결제·주문완료 전체 플로우를 구현했습니다. Zustand `cartStore`(persist)로 로컬 장바구니 상태를 관리하고, `/api/cart`로 상품 정보를 일괄 조회하는 방식으로 서버 연동을 연결했습니다. TossPay·KakaoPay 결제 연동, 주문완료 확인 페이지, 마이페이지 주문내역 페이지도 함께 추가했습니다. 아울러 Supabase 스키마 캐시 누락으로 발생하던 상품 목록 API 500 에러를 수정했습니다.

## 세부 작업

### 1. 장바구니·주문 API 라우트 추가 (`app/api/cart/`, `app/api/order/`, `app/api/orders/`)

- `GET /api/cart?ids=1,2,3` — 장바구니 상품 ID 목록으로 상품 정보(이름·가격·재고·판매자) 일괄 조회. 최대 50개 제한
- `POST /api/order` — 재고 검증 → 주문 생성 → `order_items` 삽입 → `products.stock`·`sales_count` 업데이트를 단일 트랜잭션으로 처리
- `GET /api/order/[orderId]` — 단건 주문 상세 조회 (결제완료 페이지용)
- `GET /api/orders` — 내 주문 목록, 상태 필터 + 페이지네이션. 탭 `배송중`은 UX상 `배송준비` 포함
- `lib/shipping.ts` — 배송비 로직 분리. 3만 원 미만 3,000원, 이상 무료

### 2. 장바구니 상태 관리 (`store/cartStore.ts`, `hooks/useAddToCart.ts`)

- Zustand `persist`로 `localStorage`에 상품 ID·수량 유지 (`cookeat-cart` 키)
- `cachedProducts` / `cachedProductsKey`로 API 재요청 방지 — ID 집합이 동일하면 캐시 재사용
- `checkoutItems`로 결제 단계에서 선택된 항목만 분리 전달
- `useAddToCart` — 전역 훅으로 추출. `addItems` 호출 후 toast 알림

### 3. 장바구니 컴포넌트 리팩토링

- `useCartItems` — API fetch + 캐시 관리, 병합된 상품 목록, 배송비 계산을 단일 훅으로 추상화. `cancelled` 플래그로 언마운트 경쟁 조건 방지
- `useCartSelection` — 전체/개별 체크박스 상태, 선택 합계 금액 계산 분리
- `QtyControl` — 수량 증감 UI 컴포넌트 추출
- `CartPaymentSummary` — 선택 금액·배송비·최종 합계 표시 컴포넌트 분리
- `OrderList.tsx` 삭제 — 역할이 `CartList` + `CartItemRow`로 흡수됨

### 4. 결제·주문완료 플로우 (`checkout/`, `complete/`)

- `useCheckoutPayment` — KakaoPay는 `POST /payment/kakao/ready` → redirect, TossPay·카드는 Toss SDK v1 동적 로드 후 `requestPayment`. KakaoPay redirect URL hostname 화이트리스트 검증
- `usePaymentConfirm` — 결제완료 페이지 마운트 시 `paymentKey`·`orderId`·`amount` 쿼리 파라미터 검증 → `POST /payment/toss/confirm` 호출 → 주문 상세 표시

### 5. 마이페이지 주문내역 (`app/(main)/mypage/orders/`, `components/(auth)/mypage/orders/`)

- `OrderStatusTabs` — 전체·진행중·배송완료·취소 탭 필터
- `OrderCard` / `OrderCardActions` — 주문 카드 및 상태별 액션(구매확정·반품신청 등) 분리
- `OrderPagination` — 페이지네이션 UI

### 6. products API 버그 수정 (`app/api/products/route.ts`)

- **원인:** `ingredients ( category )` Supabase JOIN 구문이 스키마 캐시에 FK가 없어 500 에러 발생
- SELECT에서 `ingredients ( category )` 제거 → `ingredient_id` 직접 선택
- 리뷰 평점 집계와 함께 `Promise.all`로 ingredient 카테고리 별도 조회, Map으로 매핑

### 7. next/image 허용 호스트 추가 (`next.config.ts`)

- `thumb.cjonstyle.net` — 상품 이미지 외부 호스트 등록
