9차 코드 리뷰 피드백을 반영했습니다. `products.ingredient_id` 컬럼이 실제 DB에 존재하지 않아 상품 상세(`/shopping/[id]`)가 전부 404로 막히고 목록 API도 500을 반환하던 문제를 수정했습니다. 실제 스키마(`products.category_id → categories → ingredients`)에 맞게 쿼리를 재구성하고, `CategoryName` 타입과 필터 UI 카테고리 목록도 DB 실값으로 전면 정렬했습니다. 실DB 연결 이후 더 이상 쓰이지 않던 `mockProducts.ts`도 제거했습니다.

## 세부 작업

### 1. 상품 상세 404 수정 (`lib/products.ts`)

- **원인:** `getProductDetail` select에 `ingredients ( ingredient_id, category )` 조인이 있었으나, 실제 DB의 `products`에는 `ingredient_id` 컬럼이 없고 `category_id → categories` FK만 존재함. PostgREST가 PGRST200(관계 없음) 에러를 반환하고 `if (error || !product) return null` 분기에 걸려 모든 상세 페이지가 notFound()로 처리됨
- `ingredients ( ingredient_id, category )` → `categories ( category_id, name )` 조인으로 교체
- `IngredientRow` 로컬 타입 → `CategoryRow`로 변경, 반환 매핑 업데이트
- `ProductDetail.ingredientId`는 실제 DB에 대응 컬럼이 없고 화면에서도 미사용임을 확인 후 타입에서 제거

### 2. 상품 목록 500 수정 (`app/api/products/route.ts`)

- **원인:** select 문자열에 `ingredient_id` 컬럼 포함 → `column products.ingredient_id does not exist` (42703) 에러
- 상품 select: `ingredient_id` → `category_id`
- 카테고리 필터: `ingredients.eq(category)` → `ingredients.eq(category)`로 `parent_id` 조회 후 `categories.in(parent_id)` 경유 `category_id` 목록 확보
- 대카테고리명 역조회: `ingredients` 단독 쿼리 → `categories.select('category_id, ingredients(category)')` 중첩 조인으로 교체
- 변수명 정리: `ingredientResult` → `parentCategoryResult`, `ingredientIds` → `parentIds`, `ingredientIds2` → `categoryIds`
- 불필요 컬럼 제거: `parent_id`, `ingredient_id` from categories select

### 3. `CategoryName` 타입 DB 실값 반영 (`types/ingredient.ts`)

- 기존 타입 값(과일·견과, 정육·계란, 수산·해산물, 쌀·잡곡, 오일/소스, 밀키트)이 DB `ingredients.category` 실값과 불일치
- DB 조회로 실값 확인 후 전면 교체

| 기존 | 실값(DB) |
|---|---|
| 과일·견과 | 과일·견과·쌀 |
| 정육·계란 | 정육·가공육·달걀 |
| 수산·해산물 | 수산·해산물·건어물 |
| 쌀·잡곡 | (과일·견과·쌀로 흡수) |
| 오일/소스 | 면·양념·오일 |
| 밀키트 | 베이커리 |

### 4. 필터 UI 카테고리 동기화 (`FilterCategory.tsx`)
