# 2026-06-10 작업 내역

## 오늘 반영된 주요 작업 요약

주문/결제 페이지 전체 컴포넌트를 구현하고, 카카오페이·신용카드·토스페이 실결제 API를 연결했습니다. 결제 로직은 `useCheckoutPayment` 훅으로 분리하고, 결제 완료 페이지에서 각 PG사 승인 처리를 수행합니다.

## 세부 작업

### 1. 주문/결제 페이지 컴포넌트 구현

- **파일 구성**
  - `DeliveryInfo.tsx` — 배송지 목록 라디오 선택, 새 배송지 추가 버튼
  - `OrderItems.tsx` — 주문 상품 확인 목록
  - `PaymentMethod.tsx` — 카드·카카오·토스·무통장·휴대폰 5종 선택 그리드, 카드 선택 시 카드사·할부 드롭다운 노출
  - `OrderAgreement.tsx` — 전체 동의 / 개별 필수 동의 체크박스, 전부 체크 시 결제 버튼 활성화
  - `PaymentSummary.tsx` — 상품금액·할인·쿠폰·배송비·최종금액·적립예정 포인트 표시, checkout/complete 두 모드 지원
  - `CheckoutIcons.tsx` — 결제 수단 아이콘 SVG 및 카카오페이·토스페이 브랜드 이미지
  - `Checkout.tsx` — 위 컴포넌트를 조합, `allAgreed` / `paymentMethod` 상태 관리

- **상태 끌어올리기**
  - `OrderAgreement` → `Checkout`: `allAgreed` 상태 콜백으로 전달, 결제 버튼 disabled 제어
  - `PaymentMethod` → `Checkout`: 선택 수단 콜백으로 전달, 결제 핸들러에서 분기

---

### 2. 결제 로직 훅 분리 (`useCheckoutPayment.ts`)

- **원인:** `Checkout.tsx`가 122줄을 초과해 결제 로직을 별도 훅으로 분리
- `loadTossV1()` — Toss v1 스크립트를 동적 로드하는 내부 헬퍼 (중복 로드 방지)
- `useCheckoutPayment(paymentMethod)` — 선택 수단별 분기 처리
  - `kakao` → `/api/payment/kakao/ready` 호출 후 리다이렉트, `tid` / `orderId` sessionStorage 저장
  - `card` → Toss `requestPayment('카드')`
  - `toss` → Toss `requestPayment('토스페이')`
  - `bankbook` / `mobile` → "준비 중" 안내 alert

---

### 3. 카카오페이 결제 API 라우트

- `POST /api/payment/kakao/ready`
  - KakaoPay Open API에 결제 준비 요청
  - `tid`, `next_redirect_pc_url`, `next_redirect_mobile_url` 반환
  - `TC0ONETIME` CID + DEV 시크릿 키로 테스트 결제 가능

- `POST /api/payment/kakao/approve`
  - 결제 완료 후 리다이렉트로 전달된 `pg_token`으로 최종 승인
  - sessionStorage에서 `tid` / `orderId` 복원 후 승인 요청

- **트러블슈팅:** 초기 PRD 키 → DEV 키로 교체, 카카오 개발자 콘솔에서 Web 플랫폼에 `http://localhost:3000` 등록 후 -403 해소

---

### 4. 토스 결제 confirm API 라우트

- `POST /api/payment/toss/confirm`
  - 신용카드·토스페이 공용 결제 승인 처리
  - `paymentKey`, `orderId`, `amount`를 받아 Toss 서버에 최종 승인 요청
  - Authorization: `Basic base64(secretKey:)`

- **트러블슈팅**
  - 공유 테스트 키(`test_ck_D5GePWvyJnrK0W0k6q8gLzN97Eom`)는 외부 도메인에서 401 — 개인 발급 테스트 키로 교체
  - v2 SDK 사용 시 UNKNOWN_ERROR — Toss v1 스크립트(`js.tosspayments.com/v1/payment`) 동적 로드 방식으로 전환

---

### 5. 결제 완료 페이지 구현

- `Complete.tsx` — URL 쿼리 파라미터로 결제 수단 판별 후 승인 API 호출
  - `paymentKey` 존재 시 → Toss confirm
  - `pg_token` 존재 시 → Kakao approve
- `PaymentResult.tsx` — loading / success / fail 상태별 UI 분리
  - success: 체크 아이콘 + 홈으로 가기 + PaymentSummary
  - fail: 다시 시도하기 버튼
