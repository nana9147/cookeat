### 2. 결제 로직 훅 분리 (`useCheckoutPayment.ts`)

- **원인:** `Checkout.tsx`가 122줄을 초과해 결제 로직을 별도 훅으로 분리
- `loadTossV1()` — Toss v1 스크립트를 동적 로드하는 내부 헬퍼 (중복 로드 방지)
- `useCheckoutPayment(paymentMethod)` — 선택 수단별 분기 처리
  - `kakao` → `/api/payment/kakao/ready` 호출 후 리다이렉트, `tid` / `orderId` sessionStorage 저장
  - `card` → Toss `requestPayment('카드')`
  - `toss` → Toss `requestPayment('토스페이')`
  - `bankbook` / `mobile` → "준비 중" 안내 alert

### 3. 카카오페이 결제 API 라우트

- `POST /api/payment/kakao/ready`
  - KakaoPay Open API에 결제 준비 요청
  - `tid`, `next_redirect_pc_url`, `next_redirect_mobile_url` 반환
  - `TC0ONETIME` CID + DEV 시크릿 키로 테스트 결제 가능

- `POST /api/payment/kakao/approve`
  - 결제 완료 후 리다이렉트로 전달된 `pg_token`으로 최종 승인
  - sessionStorage에서 `tid` / `orderId` 복원 후 승인 요청

- **트러블슈팅:** 초기 PRD 키 → DEV 키로 교체, 카카오 개발자 콘솔에서 Web 플랫폼에 `http://localhost:3000` 등록 후 -403 해소

---

### 4. 토스 결제 confirm API 라우트

- `POST /api/payment/toss/confirm`
  - 신용카드·토스페이 공용 결제 승인 처리
  - `paymentKey`, `orderId`, `amount`를 받아 Toss 서버에 최종 승인 요청
  - Authorization: `Basic base64(secretKey:)`

- **트러블슈팅**
  - 공유 테스트 키(`test_ck_D5GePWvyJnrK0W0k6q8gLzN97Eom`)는 외부 도메인에서 401 — 개인 발급 테스트 키로 교체
  - v2 SDK 사용 시 UNKNOWN_ERROR — Toss v1 스크립트(`js.tosspayments.com/v1/payment`) 동적 로드 방식으로 전환

### 9. API 클라이언트 fetch → axios 통일

- **원인:** `lib/api.ts`에 axios 인스턴스가 정의되어 있었으나 실제로 쓰는 곳이 없고, 각 훅·컴포넌트에서 `fetch`를 직접 호출하는 혼재 상태
- 아래 파일 전체를 `api` 인스턴스 호출로 교체
  - `hooks/user/useSellerApply.ts`, `useAvatarUpload.ts`, `useProfileEditForm.ts`, `useUserInfo.ts`
  - `components/(auth)/mypage/MypageCategory.tsx`
  - `components/(auth)/order/checkout/useCheckoutPayment.ts`, `complete/Complete.tsx`
- `Authorization` · `Content-Type` 헤더는 인터셉터가 자동 처리하므로 각 호출에서 제거
- `lib/api.ts` 에러 인터셉터에 `response.data` 보존 추가 — `useSellerApply`의 필드별 validation 에러(`errors`) 접근을 위해

#### ESLint 수정 (함께 처리)

- `Complete.tsx` — `useEffect` 내 동기 `setState` 제거, `paymentKey`·`pgToken`·`orderId`·`amount`를 effect 밖으로 추출해 초기 state 계산 및 deps 배열에 사용
- `useProfileEditForm.ts` — `useEffect`로 store 값을 로컬 state에 동기 복사하던 패턴 제거, `nicknameOverride`·`phoneOverride` nullable state로 교체 (값 없으면 서버값 그대로 표시)
- `useUserInfo.ts` — store `user`를 로컬 state로 복사하던 첫 번째 `useEffect` 제거, API 응답을 단일 `apiProfile` state로 관리하고 최종값은 `apiProfile ?? user` 순으로 파생
- `useSellerApply.ts` — `fetchApplication`을 `useCallback([accessToken])`으로 감싸 exhaustive-deps 해소
