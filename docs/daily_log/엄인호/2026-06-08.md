# 2026-06-08 작업 내역

## 오늘 반영된 주요 작업 요약

소셜 재로그인 시 프로필 사진이 공급자 기본 이미지로 초기화되는 버그를 수정하고, 프로필 이미지 조회 로직을 서버 API 기반으로 개선했습니다. 이후 장바구니 페이지 UI를 디자인 시안에 맞게 구현했습니다.

---

## 세부 작업

### 1. 소셜 재로그인 시 프로필 사진 초기화 버그 수정

- **원인:** 소셜 로그인 시 Supabase가 OAuth 공급자(Google·Kakao) 데이터로 `user_metadata.avatar_url`을 덮어씀. 직접 업로드한 아바타가 이 키에 저장돼 있어 재로그인마다 공급자 이미지로 리셋되는 문제
- `app/api/user/avatar/route.ts` — 업로드 URL 저장 키를 `avatar_url` → `custom_avatar_url`로 변경 (OAuth가 건드리지 않는 독립 키)
- `app/api/auth/profile/route.ts` — 응답에 `profileImage` 추가, `custom_avatar_url → avatar_url` 우선순위로 조회
- `services/auth/authTypes.ts` — `toAuthUser`에서 동일 우선순위 적용
- `app/auth/callback/page.tsx` — 소셜 콜백 `setAuth` 시 동일 우선순위 적용
- `components/(auth)/AuthInitializer.tsx` — 토큰 갱신 이벤트에서 동일 우선순위 적용

### 2. 프로필 이미지 조회 로직 개선

- **원인:** `profileImage`가 Zustand 스토어의 `user.profileImage`에만 의존해, 재로그인 후 JWT에 stale 값이 담기면 표시가 깨지는 문제
- `hooks/user/useUserInfo.ts` — `profileImage`를 로컬 state로 분리, 닉네임·전화번호처럼 `/api/auth/profile` GET에서 최신값을 받아오도록 수정. `patchUser` 직후 즉시 반영을 위해 `user?.profileImage` 변경도 감지

- `components/(auth)/order/Cart.tsx` — 전체선택 체크박스 헤더 (클라이언트 상태 관리) + CartList + RecommendedItems 조합

- `components/(auth)/order/RecommendedItems.tsx` — 함께 사면 좋아요 상품 그리드
- `components/(auth)/order/SellerInfo.tsx` — 판매자별 무료배송 기준 안내 박스
- `components/(auth)/order/Orderpay.tsx` — 총 결제금액·적립금·주문하기 버튼·배송 안내

### 4. 장바구니 라우트 변경 (/order → /cart)

- **원인:** seller 페이지에서 /order 라우트를 사용할 예정이라 충돌 방지를 위해 장바구니 라우트를 /cart로 변경
- `app/(main)/order/` → `app/(main)/cart/`로 폴더 이전
- `CartStepper.tsx`, `Orderpay.tsx`, `HeaderActions.tsx` 내 경로 참조 일괄 수정

### 5. 장바구니 페이지 반응형 레이아웃 적용

- **기준:** mobile-first (기본 = 320px~), `tablet:` (768px~), `desktop:` (1440px~)
- `Order.tsx` — 모바일 단일 컬럼 / desktop 2컬럼 레이아웃 (우측 결제 요약 `sticky` 고정)
- `CartItemRow.tsx` — 모바일은 수량·가격을 상품 정보 하단에 표시, 태블릿+는 인라인 배치
- `RecipeGroupCard.tsx` — 좌측 영역 `min-w-0` + 레시피명 `truncate`로 320px 대응, 배지 행 `flex-wrap`
- `Cart.tsx` — 헤더 320px overflow 방지 (삭제 버튼 `text-xs` 모바일, `shrink-0`)
- `CartList.tsx` — 패딩 `px-4 tablet:px-5` 적용
- `RecommendedItems.tsx` — `grid-cols-4` → `grid-cols-2 tablet:grid-cols-4`

### 6. 닉네임 중복 DB 제약 추가

- **원인:** 1차 코드 리뷰에서 `check-nickname` API와 회원가입 API 사이에 race condition 가능성 지적 — DB 레벨 UNIQUE 제약이 없으면 동시 가입 시 중복 닉네임이 생길 수 있음
- Supabase 대시보드 SQL Editor에서 `users` 테이블 `nickname` 컬럼에 UNIQUE 제약 추가

```sql
ALTER TABLE users ADD CONSTRAINT users_nickname_key UNIQUE (nickname);
```

- 이제 닉네임 중복은 API 레이어와 무관하게 DB가 최종 차단함

---

### 7. auth 라우트 서버 입력 검증 추가

- **원인:** 1차 코드 리뷰에서 login·register API에 서버 사이드 입력 검증이 없다는 지적 — 클라이언트 검증만으로는 악의적 요청을 막을 수 없음
- `lib/validators.ts` 신규 생성 — 서버·클라이언트 어디서나 재사용 가능한 `validateSignup` / `validateLogin` 함수 (이메일·비밀번호·닉네임·전화번호 형식 검사)
- `app/api/auth/register/route.ts` — `validateSignup` 적용, 유효하지 않은 입력 시 400 반환
- `app/api/auth/login/route.ts` — 빈 이메일·비밀번호 사전 검증 추가, 세미콜론 스타일 통일
