# 2026-07-02 작업 내역

## 오늘 반영된 주요 작업 요약

상품·레시피 상세 페이지의 리뷰 섹션을 목업 데이터에서 실제 API 연동으로 전면 교체했습니다. 리뷰 작성·수정·삭제(CRUD)와 이미지 첨부, 평점 분포 집계, 페이지네이션을 지원하는 API를 구현하고, 이를 사용하는 공용 리뷰 컴포넌트 세트(`ReviewSectionClient`, `ReviewWriteModal` 등)를 새로 만들었습니다.

마이페이지 주문 목록에서는 배송완료 주문의 상품별로 리뷰 작성 모달을 바로 띄울 수 있도록 연동했습니다. 아울러 로그인/토큰 갱신 응답에 DB `user_id`를 포함시키고, 프로필 이미지 업로드 시 `users` 테이블도 함께 갱신하도록 인증 관련 버그를 수정했습니다.

추가로 쿠폰 재사용 방지 로직을 `user_coupons.used_at` 기준으로 바로잡고, 카카오페이 결제 요청 시 클라이언트가 보낸 금액을 그대로 신뢰하던 취약점을 서버 재검증 방식으로 수정했습니다.

---

## 세부 작업

### 1. 리뷰 조회 API 개편 (`GET /api/products/:id/reviews`, `GET /api/recipes/:id/reviews`)

- 기존에는 `users` 테이블을 조인해 닉네임만 가져왔으나, 별도 쿼리로 작성자 목록을 조회해 `authId`·`profileImage`까지 함께 반환하도록 변경
- 닉네임이 없는 경우 이메일 앞부분을 fallback으로 사용
- 전체 리뷰의 평점을 별도로 집계해 `averageRating`, `ratingBreakdown`(1~5점 분포)을 응답에 추가
- 페이지네이션(`page`, `limit`)은 기존 로직 유지, 응답 필드에 `totalCount` 포함

### 2. 리뷰 작성/수정/삭제 API

- `POST /api/products/:id/reviews`, `POST /api/recipes/:id/reviews`: 이미지 URL 검증(Supabase Storage 경로 접두사 확인, 최대 5개) 로직 정리
- 신규 `PATCH /DELETE /api/products/:id/reviews/:reviewId`, `.../recipes/:id/reviews/:reviewId` 라우트 추가
  - 작성자 본인 확인 후 수정/삭제 처리
  - 수정 시 기존 첨부 이미지를 삭제하고 새 이미지로 교체

### 3. 공용 리뷰 컴포넌트 구조화 (`components/common/`)

- `ReviewSectionClient`: 리뷰 목록 조회 훅(`useReviewSection`)을 사용해 요약·목록·작성/수정 모달을 묶어주는 클라이언트 컨테이너
- `ReviewSectionHeader`, `ReviewBody`: 헤더(작성 버튼·전체보기)와 목록 렌더링 분리
- `ReviewCard`: 작성자 아바타(`ReviewAvatar`)·첨부 이미지(`ReviewImages`) 표시, 본인 리뷰일 때 수정/삭제 버튼 노출
- `ReviewWriteModal` + `StarRatingInput`: 작성/수정 겸용 모달, `useReviewWriteForm` 훅에서 제출 로직 처리
- 기존 `ReviewSection.tsx`는 재수출(barrel) 역할로 축소하고 실제 컨테이너는 `ReviewSectionClient`로 이전

### 4. 상품/레시피 상세 페이지 연동

- `ProductTabs`, `ProductInfoTab`, `InfoTabReviewPreview`: 목업 리뷰(`mockReviews.ts`) 제거, `ReviewSectionClient`로 교체 (상품 정보 탭엔 `compact` 미리보기, 리뷰 탭엔 전체 목록)
- 레시피 상세(`recipes/[id]/page.tsx`): `ReviewSection` → `ReviewSectionClient(type="recipe")`로 교체, `mockRecipes.ts`의 `reviews` 목업 필드 제거

### 5. 마이페이지 주문 리뷰 작성 연동

- `OrderCardActions`: 배송완료 주문의 "리뷰 작성" 버튼을 활성화하고 `OrderReviewModal` 오픈
- `OrderReviewModal` + `useOrderReviewModal`: 주문 상세(`GET /api/orders/:orderId`)를 불러와 상품별 리뷰 작성 행(`OrderItemReviewRow`) 나열, 작성 완료 상태 로컬 추적
- `GET /api/orders/:orderId` 응답에 `items[].productId` 추가 (리뷰 작성 대상 상품 식별용)
- 주문 상품 리뷰는 `orderItemId` 기준으로 구매 확인 후 작성되며 포인트(10P) 자동 적립

### 6. 인증/프로필 버그 수정

- `POST /api/auth/login`, `/api/auth/refresh`: 응답에 `users.user_id`(`dbUserId`)를 포함하도록 수정, `AuthUser` 타입에 `dbUserId` 필드 추가
- `POST /api/user/avatar`: Supabase Auth 메타데이터만 갱신하고 `users.profile_image`는 갱신하지 않던 문제 수정 — 두 곳을 병렬로 함께 업데이트
- 업로드한 이미지 URL에 캐시버스팅 쿼리(`?t=timestamp`)를 붙여 브라우저 캐시로 인해 이전 프로필 이미지가 계속 보이던 문제 수정

### 7. 쿠폰 재사용 방지 로직 수정

- `GET /api/coupons`, `GET /api/coupons/validate`: 쿠폰 사용 여부를 `orders.status !== '취소'` 기준으로 판별하던 로직을 `user_coupons.used_at` 존재 여부로 변경 — 주문을 취소해도 쿠폰이 계속 "사용됨"으로 남던 반대 상황과, 반대로 재사용 가능해지던 문제를 모두 해결
- `GET /api/coupons/mine`: 응답에 쿠폰 `code` 필드 추가
- 쿠폰 식별 키를 `code`에서 `userCouponId`로 통일 (`CouponModal`, `DiscountSection`, `Checkout`, `useCheckoutPayment`) — 동일 코드의 쿠폰이 여러 장 발급된 경우를 구분할 수 있도록 함
- `app/api/seller/orders/[orderId]/db.ts`: 주문-쿠폰 조인을 `orders.coupon_id` 대신 `user_coupons` 경유로 변경

### 8. 카카오페이 결제 금액 변조 취약점 수정

- `POST /api/payment/kakao/ready`: 클라이언트가 전달한 `totalAmount`를 그대로 신뢰해 카카오에 결제 요청하던 로직 제거. 서버에서 `orderId`로 주문을 재조회해 `final_amount`를 결제 금액으로 사용하고, 주문 소유자 확인 및 `status === '결제전'` 검증 추가
- `POST /api/payment/kakao/approve`: 카카오가 승인한 실제 결제 금액(`kakaoData.amount.total`)과 서버 주문 금액(`order.final_amount`)이 다르면 결제완료 처리를 막도록 검증 추가
