## 세부 작업

### 1. sellers 테이블 스키마 수정

- **원인:** `is_approved (BOOLEAN)`과 `approve_status (ENUM)` 두 컬럼이 중복 역할을 하고 있었고, 거절 사유를 저장할 컬럼이 없었음
- `is_approved` 컬럼 제거
- `rejected_reason VARCHAR(200) NULL` 컬럼 추가 — `rejected` 상태일 때만 값 존재
- `business_address VARCHAR(300) NULL` 컬럼 추가 — 사업장 주소

```sql
ALTER TABLE public.sellers DROP COLUMN IF EXISTS is_approved;
ALTER TABLE public.sellers ADD COLUMN IF NOT EXISTS rejected_reason VARCHAR(200);
ALTER TABLE public.sellers ADD COLUMN IF NOT EXISTS business_address VARCHAR(300);
```

### 2. API 명세 수정 (`docs/api/`)

#### 06-seller.md

- `POST /seller/apply` 엔드포인트 추가 — 일반 회원(`role: user`)이 판매자 신청하는 엔드포인트
- `GET /seller/me` 응답 필드 수정: `isApproved (boolean)` → `approveStatus (string)` + `rejectedReason (string|null)`
- 엔드포인트 목록 인증 컬럼에 `(user)` / `(seller)` 구분 추가

#### 07-admin.md

- `GET /admin/sellers` 응답에서 `isApproved (boolean)` 제거 — `approveStatus` 하나로 통일
- `PATCH /admin/sellers/:sellerId/approve` 요청 바디 수정: `approved: boolean` → `status: 'approved' | 'rejected'`

#### 08-schema.md

- sellers 테이블 정의 반영: `is_approved` 제거, `rejected_reason` · `business_address` 추가
