# 2026-06-29 작업 내역

## 오늘 반영된 주요 작업 요약

마이페이지 고객센터(1:1 문의) 기능과 알림 설정 페이지를 추가했습니다. 문의 목록·작성·상세 조회를 위한 `/api/inquiries` API 라우트를 구현하고, 마이페이지 UI 컴포넌트(`InquiryList`, `InquiryForm`)를 함께 작성했습니다. 또한 마이페이지에서 사용하는 북마크·포인트·내 레시피 목록 API를 `/api/users/me/` 하위에 추가해 사용자 관련 API를 체계화했습니다.

이후 고객센터 답변 표시 버그(답변 있어도 "답변대기"로 표시), 주문 상세 모달, 전화번호 입력 라운드 누락 등 UI/기능 버그를 수정했습니다. 찜 기능을 전면 활성화하여 레시피 북마크와 상품 위시리스트 API·훅·UI를 구현하고, 마이페이지 찜 목록 재료쇼핑 탭도 연결했습니다. 마지막으로 헤더 통합 검색 기능을 추가해 레시피·재료쇼핑 결과를 한 페이지에서 확인할 수 있도록 `/search` 페이지를 구성했습니다.

## 세부 작업

### 1. 마이페이지 고객센터 페이지

- `/mypage/support` 신규 생성
- `InquiryList` 컴포넌트: 문의 목록 조회·로딩 스켈레톤·빈 상태 처리
- `InquiryForm` 컴포넌트: 문의 유형(주문·상품·배송·기타) 선택 후 제목·내용 입력 및 POST 제출
- `/mypage/support/[inquiryId]` 상세 페이지 — 채팅 버블 UI로 문의·답변 표시

### 2. 마이페이지 알림 설정 페이지

- `/mypage/alarm` 신규 생성
- 기능 준비 중 안내 UI 표시

### 3. 문의 API

- `GET /api/inquiries`: 로그인 유저의 문의 목록 조회 (답변 여부 포함)
- `POST /api/inquiries`: 새 문의 등록 (카테고리 유효성 검사 포함)
- `GET /api/inquiries/[inquiryId]`: 문의 상세 및 답변 조회
- `DELETE /api/inquiries/[inquiryId]`: 본인 문의 삭제 (답변 완료 건 삭제 불가)

### 4. 유저 마이페이지 API 정비

- `GET /api/users/me/bookmarks`: 북마크한 레시피 목록 페이지네이션
- `GET /api/users/me/points`: 포인트 적립·사용 내역 조회
- `GET /api/users/me/recipes`: 본인 작성 레시피 목록 페이지네이션

### 5. 버그 수정

- 고객센터 답변 표시 오류: Supabase UNIQUE FK 단일 객체 반환 문제 수정 (배열·객체 양쪽 처리)
- 주문 상세: 링크 대신 모달로 전환, 배송 정보 별도 쿼리(FK 중복 충돌 해결)
- 전화번호 입력 컴포넌트 라운드 스타일 누락 수정
- HeroSection 레시피 버튼 링크 누락 추가

### 6. 찜 기능 전면 활성화

- `GET/POST /api/recipes/[recipeId]/bookmark`: 레시피 북마크 상태 조회 및 토글
- `GET/POST /api/products/[productId]/wishlist`: 상품 위시리스트 상태 조회 및 토글
- `GET /api/users/me/stats`: 찜 카운트(레시피 북마크 + 상품 위시리스트) 반환
- `GET /api/users/me/wishlists`: 찜한 상품 목록 페이지네이션
- `useBookmark`, `useWishlist` 훅 신규 작성
- RecipeCard·RecipeBookmarkButton·IngredientCardImage·ProductPurchasePanel 찜 버튼 API 연동
- 마이페이지 찜 목록 재료쇼핑 탭 구현
- 마이페이지 유저카드 찜 배지에 레시피+상품 합산 카운트 표시
- 찜 목록 그리드 반응형(2→3→4열) 수정

### 7. 코드 리뷰 반영

- `app/api/auth/login/route.ts`: role 화이트리스트(`['user','seller','admin']`) 검증 추가 — 예상 외 값이 권한으로 저장되는 것 방지
- `PointHistory.tsx`: `.catch(() => {})` 에러 묵살 제거 → "포인트 내역을 불러오지 못했습니다" 에러 메시지 표시
- `CancelList.tsx`: 동일하게 에러 상태 추가
- `LikesList.tsx`: 레시피·상품 탭 각각 별도 에러 상태 추가
- `app/api/products/[productId]/wishlist/route.ts`: insert/delete 에러를 무시하고 항상 200 반환하던 버그 수정

### 8. 헤더 통합 검색

- 헤더 검색창: 검색어 입력 후 `/search?keyword=...` 이동
- `/search` 페이지: 레시피·재료쇼핑 결과를 섹션별로 동시 표시, 각 "전체보기" 링크
- `/recipes`, `/shopping` 페이지가 `keyword` 쿼리 파라미터로 필터링 지원
