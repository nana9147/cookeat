# 2026-06-30 작업 내역

## 오늘 반영된 주요 작업 요약

주문/결제 페이지에 배송지 저장 및 관리 기능을 추가했습니다. 유저가 여러 배송지를 등록하고 기본 배송지를 선택해 결제 시 바로 사용할 수 있도록 CRUD API와 UI 컴포넌트를 구현했습니다. react-daum-postcode를 활용한 우편번호 검색과 PhoneInput 기반 전화번호 자동 포맷(010-0000-0000)도 함께 적용했습니다.

아울러 마이페이지 주문 목록에 결제 미완료(결제전) 주문이 노출되는 버그를 수정하고, 결제 버튼 이중 클릭으로 주문이 중복 생성되는 문제도 차단했습니다. 결제 흐름에서 사용자가 PG 창을 닫거나 취소해도 pending 주문이 목록에 남지 않도록 서버 필터를 추가했습니다.

## 세부 작업

### 1. 배송지 관리 API

- `GET /api/users/me/addresses`: 로그인 유저의 배송지 목록 조회
- `POST /api/users/me/addresses`: 새 배송지 등록 (첫 번째는 자동 기본 설정)
- `PATCH /api/users/me/addresses/[id]`: 배송지 수정, 기본 배송지 변경
- `DELETE /api/users/me/addresses/[id]`: 배송지 삭제 (기본 배송지 삭제 시 다음 주소로 자동 승격)
- `requireAuth` 가드 + 소유권 검증, 전화번호 형식 서버 검증(`validatePhone`)

### 2. 배송지 UI (주문/결제 페이지)

- `DeliveryInfo`: 배송지 목록 표시, 선택된 배송지를 부모로 전달
- `DeliveryAddressCard`: 배송지 카드 (라디오 선택, 수정/삭제)
- `AddressFormModal`: 배송지 추가/수정 모달
- `AddressSearch`: react-daum-postcode 우편번호 검색 → 기본주소 자동 입력
- `AddressLabelPicker`: 집/회사/직접입력 프리셋 선택
- `AddressContactFields`: 수령인 + 연락처(PhoneInput)
- `useUserAddresses`: 배송지 CRUD 훅

### 3. 전화번호 포맷 및 검증

- `AddressContactFields`에 `PhoneInput` 적용 — 입력 시 010-XXXX-XXXX 자동 포맷
- `lib/validators.ts`에 `validatePhone()` 추가, POST/PATCH 라우트에서 형식 검증

### 4. 버그 수정

- **결제전 주문 목록 노출**: `GET /api/orders`에 `.neq('status', '결제전')` 필터 추가 — 결제 취소/실패로 생성된 pending 주문이 마이페이지에서 보이지 않도록 수정
- **결제 버튼 이중 클릭**: `Checkout.tsx`에 `paying` state 추가, API 호출 중 버튼 비활성화 및 "처리 중..." 표시

### 5. 컴포넌트 구조 정리 (TSX 80줄 이내)

- `CheckoutIcons.tsx` (153줄) → `AddressIcons.tsx` + `CheckoutIcons.tsx` 분리
- `DeliveryAddressCard.tsx` (110줄) → `AddressCardInfo.tsx` 추출
- `AddressSearch.tsx` (91줄) → `PostcodePanel.tsx` 추출
- `AddressFormModal.tsx` (134줄) → `AddressFormFields.tsx`, `AddressContactFields.tsx` 추출
