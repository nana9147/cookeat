# 2026-06-15 작업 내역

## 오늘 반영된 주요 작업 요약

장바구니 플로우(장바구니 → 주문서 작성 → 결제 완료)의 마지막 단계인 결제 완료 페이지를 구현했습니다. `components/order` 폴더를 `components/cart`로 이동하고, `PaymentSummary`를 공용 컴포넌트로 확장해 cart·checkout·complete 세 페이지에서 재사용하도록 정리했습니다.

---

## 세부 작업

### 1. components/order → components/cart 폴더 이동

- **원인:** 컴포넌트 폴더명이 라우트 구조(`app/(main)/cart/`)와 일치하지 않아 혼란 유발
- `components/(auth)/order/` 하위 전체를 `components/(auth)/cart/`로 이동
- `app/(main)/cart/page.tsx`, `checkout/page.tsx`, `complete/page.tsx`의 import 경로 일괄 수정

---

### 2. PaymentSummary 공용 컴포넌트 확장

- **원인:** checkout 전용으로 작성된 컴포넌트를 complete 페이지에서도 재사용하기 위해 확장 필요
- `noCard?: boolean` 추가 — `true`일 때 카드 래퍼·제목·안내박스 없이 금액 rows만 렌더링
  - complete 페이지 왼쪽 카드(배송정보+결제정보) 내부에 임베드할 때 사용
- `paymentMethodLabel?: string` 추가 — 결제 수단 행을 동적으로 표시
- `장바구니로 돌아가기` 링크를 `mode === 'checkout'` 조건으로 이동 (complete에서 미노출)

---

### 3. 결제 완료 페이지 구현 (`cart/complete/`)

- **구성:** 파일당 80줄 이하로 컴포넌트 분리
  - `completeData.ts` — 타입, 상수(DELIVERY, ORDER_INFO, ITEMS, FINAL_AMOUNT), PAYMENT_LABEL
  - `CompleteHeader.tsx` — 카드1: 체크마크 + 완료 메시지 + 주문번호/일시 + 주문내역 링크
  - `CompleteOrderInfo.tsx` — 카드2(좌): 배송 정보 + `PaymentSummary(noCard)` 결제 정보
  - `CompleteOrderItems.tsx` — 카드3(우): 주문 상품 목록 + 결제금액 합계 + CTA 버튼
  - `CompleteNotice.tsx` — 카드4: 안내사항
  - `PaymentResult.tsx` — 오케스트레이터 (loading/fail/success 분기 후 카드 조합)
- **레이아웃:** 카드2·3은 `flex-1` + `desktop:items-stretch`로 1:1 동일 높이 유지
- **반응형:** mobile/tablet 세로 스택, desktop 가로 2컬럼
