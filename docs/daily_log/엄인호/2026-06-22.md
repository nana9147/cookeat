# 2026-06-22 작업 내역

## 오늘 반영된 주요 작업 요약

`IngredientCard`에서 장바구니 수량 조절 로직을 `IngredientCartControl` 컴포넌트로 분리해 단일 책임 원칙을 맞췄습니다.

홈 화면 더미 데이터를 실 API로 전환했습니다. `useHomeData` 훅을 신규 작성해 `/api/recipes`·`/api/recipe-categories` 호출을 추상화하고, `CategorySection`·`PopularRecipeSection`·`TodayRecipeCard`·`UserPointCard`에 연동했습니다. 로딩 중 스켈레톤 UI 적용, `useUserInfo` 레이스 컨디션 방어도 함께 처리했습니다.

결제 흐름에서 세 가지 버그를 수정했습니다. `payment_method`에 영문 코드가 그대로 삽입돼 DB 제약을 위반하던 문제, `status: '결제전'`이 DB enum에 없어 INSERT가 실패하던 문제, `order_id`가 VARCHAR(30)을 초과(UUID 40자)해 500 에러가 발생하던 문제를 각각 수정했습니다.

쇼핑 상품 이미지 중 `encrypted-tbn0.gstatic.com` 도메인이 `next/image` 허용 목록에 없어 런타임 에러가 발생하던 문제도 수정했습니다.

---

## 세부 작업

### 1. IngredientCartControl 컴포넌트 분리

- `IngredientCard.tsx`에서 수량 조절 UI·로직을 `IngredientCartControl.tsx`로 추출
- `IngredientCard`는 렌더링, `IngredientCartControl`은 cartStore 연동만 담당

### 2. 홈 화면 API 실 데이터 연동

- `components/home/useHomeData.ts` 신규 작성 — `usePopularRecipes`, `useTodayRecipe`, `useRecipeCategories`
- `app/api/recipes/route.ts` 신규 — sort·category·limit·page 쿼리 지원
- `app/api/recipe-categories/route.ts` 신규
- `CategorySection`, `PopularRecipeSection`, `TodayRecipeCard` 훅으로 교체, 스켈레톤 적용
- `UserPointCard` 실제 닉네임·포인트 표시
- `useUserInfo` ignore 패턴으로 레이스 컨디션 방어

### 3. 결제 오류 3종 수정 (`app/api/order/route.ts`)

- **payment_method 매핑:** `PAYMENT_METHOD_MAP` 추가 — `card → 카드`, `kakao → 카카오페이`, `toss → 토스페이`, `bankbook → 무통장입금`, `mobile → 휴대폰결제`
- **status 제거:** `'결제전'`은 DB enum에 없는 값 → 필드 제거, DB default(`'결제완료'`) 사용
- **order_id 단축:** `ORD-${UUID}` (40자) → `ORD-YYYYMMDD-XXXXXXXX` (21자)로 VARCHAR(30) 이내로 변경

### 4. next.config 이미지 도메인 추가

- `encrypted-tbn0.gstatic.com`을 `remotePatterns`에 추가
