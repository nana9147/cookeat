# 2026-06-22 작업 내역

## 오늘 반영된 주요 작업 요약

`IngredientCard`에서 장바구니 수량 조절 로직을 `IngredientCartControl` 컴포넌트로 분리하여 단일 책임 원칙을 맞췄습니다.

홈 화면의 CategorySection·PopularRecipeSection·TodayRecipeCard·UserPointCard를 더미 데이터에서 실제 API로 전환했습니다. `useHomeData` 훅을 신규 작성해 세 개의 API 호출을 추상화하고, `/api/recipes`·`/api/recipe-categories` 라우트를 새로 추가했습니다. 로딩 중에는 스켈레톤 UI가 표시되도록 처리했으며, `useUserInfo`에 레이스 컨디션 방어(ignore 패턴)와 에러 핸들링을 보강했습니다.

주문 생성 API(`/api/order`)에서 두 가지 버그를 수정했습니다. 첫째, `payment_method`에 영문 코드(`'card'`, `'kakao'` 등)가 그대로 삽입되어 DB 제약 위반으로 주문이 생성되지 않던 문제를 `PAYMENT_METHOD_MAP`을 추가해 한글 값으로 변환하도록 수정했습니다. 둘째, `status: '결제전'`이 DB enum에 없는 값이어서 INSERT가 500 에러로 실패하던 문제를 해당 필드를 제거하여 DB default(`'결제완료'`)를 사용하도록 수정했습니다.

쇼핑 상품 카드에서 `encrypted-tbn0.gstatic.com` 도메인 이미지가 `next/image` unconfigured host 런타임 에러를 일으키던 문제를 `next.config.ts`에 해당 도메인을 추가하여 수정했습니다.

## 세부 작업

### 1. IngredientCartControl 컴포넌트 분리 (`components/ingredient/`)

- `IngredientCard.tsx`에서 장바구니 수량 조절 UI·로직을 `IngredientCartControl.tsx`로 추출
- `IngredientCard`는 렌더링만, `IngredientCartControl`은 수량 상태와 cartStore 연동만 담당

### 2. 홈 화면 API 실 데이터 연동

- **`components/home/useHomeData.ts`** 신규 작성
  - `usePopularRecipes`: `/api/recipes?sort=popular&limit=6` 호출
  - `useTodayRecipe`: `/api/recipes?sort=latest&limit=1` 호출
  - `useRecipeCategories`: `/api/recipe-categories` 호출
- **`app/api/recipes/route.ts`** 신규: `sort`, `category`, `limit`, `page` 쿼리 파라미터 지원
- **`app/api/recipe-categories/route.ts`** 신규: `recipe_categories` 목록 반환
- `CategorySection`, `PopularRecipeSection`, `TodayRecipeCard`: `useHomeData` 훅으로 교체, 로딩 시 스켈레톤 처리
- `UserPointCard`: `useUserInfo`로 실제 닉네임·포인트 표시
- `hooks/user/useUserInfo.ts`: ignore flag로 레이스 컨디션 방어, 에러 상태 반환 추가

### 3. paymentMethod DB 한글 매핑 및 status 버그 수정 (`app/api/order/route.ts`)

- **원인 1:** `payment_method: paymentMethod ?? '미정'`에서 영문 코드가 그대로 DB에 삽입 → DB 제약 위반으로 주문 생성 실패
  - `PAYMENT_METHOD_MAP` 상수 추가: `card → 카드`, `kakao → 카카오페이`, `toss → 토스페이`, `bankbook → 무통장입금`, `mobile → 휴대폰결제`
- **원인 2:** `status: '결제전'`이 DB ENUM(`결제완료|주문확인|배송준비|배송중|배송완료|취소`)에 없는 값 → INSERT 500 에러
  - `status` 필드 제거, DB default(`'결제완료'`) 사용

### 4. next/image 호스트 에러 수정 (`next.config.ts`)

- `encrypted-tbn0.gstatic.com` 도메인을 `remotePatterns`에 추가
