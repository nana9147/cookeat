## 오늘 반영된 주요 작업 요약

레시피 목록·상세 페이지의 목 데이터를 실제 API로 전환했습니다. 레시피 상세 API 라우트(`GET /api/recipes/[recipeId]`)를 신규 작성하고, 목록 페이지는 `/api/recipe-categories`·`/api/recipes`를 연동해 카테고리 탭·정렬·페이지네이션이 동작하도록 했습니다. 공유 타입을 `recipes/types.ts`로 분리해 컴포넌트 간 타입 일관성을 맞췄습니다.

홈 화면에서 인기 레시피 카드를 클릭하면 상세 페이지로 이동하도록 수정했고, 더보기 버튼에 `/recipes` 링크를 연결했습니다. LCP 경고 대응을 위해 TodayRecipeCard 이미지에 `priority`를 적용했습니다.

## 세부 작업

### 1. 레시피 공유 타입 분리

- `app/(main)/recipes/types.ts` 신규 작성
- `RecipeListItem`, `RecipeDetail`, `RecipeCategory`, `SortKey`, `SORT_OPTIONS` 정의
- 목록·상세 컴포넌트 전체가 이 타입을 참조하도록 변경

### 2. 레시피 상세 API 신규 작성

- `GET /api/recipes/[recipeId]` 라우트 신규 생성
- `recipes`, `users`, `recipe_categories`, `recipe_steps`, `recipe_ingredients` 조인
- `reviews` 테이블에서 rating 평균·리뷰 수 계산
- 404 / 400 에러 분기 처리

### 3. 레시피 목록 페이지 API 연동

- `RecipeClient`: `useEffect`로 카테고리·레시피 데이터 fetch, 로딩 스켈레톤 적용
- `RecipeCategoryTabs`: `/api/recipe-categories` 동적 카테고리 렌더링
- `RecipeSortBar`: `popular/latest/scrap` API 파라미터 매핑 (인기순·최신순·스크랩순)
- `RecipeGrid`: 로딩 중 스켈레톤 12칸 표시
- `RecipeCard`: `recipeId` 기반 상세 링크

### 4. 레시피 상세 페이지 API 연동

- `[id]/page.tsx`: 서버 컴포넌트에서 fetch로 API 호출, 404 시 notFound()
- `RecipeMetaRow`: DB에 없는 calories 제거 → difficulty(난이도) 카드로 교체
- `RecipeAuthor`: 프로필 이미지 지원
- `RecipeIngredientItem`: 연결 상품 없을 때 가격·장바구니 버튼 미표시
- `RecipeStepItem`: 조리 단계 이미지 표시 지원

### 5. 홈 화면 수정

- `PopularRecipeSection` 더보기 버튼 → `/recipes` Link 교체
- `RecipeCard(home)` `recipeId` prop 추가, 클릭 시 `/recipes/:id` 이동
- `TodayRecipeCard` Image `priority` 적용, 조건부 렌더링 제거 (LCP 대응)
- `EventBanner` 문구 수정

## 세부 수정

### 1. ReviewSection ratingBreakdown 실 데이터 연결

- `reviews` 테이블에서 별점 분포를 집계해 `ratingBreakdown: Record<1|2|3|4|5, number>` 반환
- `RecipeDetail` 타입에 `ratingBreakdown` 필드 추가
- 기존에 `{ 5:0, 4:0, 3:0, 2:0, 1:0 }` 하드코딩이었던 것을 실 데이터로 교체

### 2. Server Component HTTP 자기 호출 제거

- `lib/serverRecipes.ts` 신규 추출 (`server-only` 선언)
- Supabase 조회 로직을 `fetchRecipeDetail()` 함수로 분리
- 레시피 상세 페이지(`[id]/page.tsx`)가 `fetch('/api/recipes/...')` 대신 `fetchRecipeDetail()`을 직접 호출하도록 변경 (불필요한 네트워크 왕복 제거)
- API 라우트(`route.ts`)도 동일 함수를 재사용하도록 단순화

### 3. dangerouslyAllowSVG에 CSP 추가

- `next.config.ts`에 `contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;"` 추가
- 사용자 업로드 SVG를 통한 XSS 가능성 차단

### 4. RecipeClient 에러 상태 추가

- `error` 상태를 추가해 레시피 fetch 실패 시 "레시피를 불러오지 못했습니다. 잠시 후 다시 시도해주세요." 메시지 표시
- 기존에는 에러를 무음 처리해 빈 화면만 출력됐던 문제 해결

### 5. EventBanner 문구 수정

- `"지금 가입하고 포인트 / 2,000을"` → `"지금 가입하고 / 포인트 2,000을"` 으로 줄바꿈 위치 수정
