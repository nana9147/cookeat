## 코드 리뷰 피드백 반영 2차

`lib/serverRecipes.ts`에 대한 추가 코드 리뷰 피드백 4개 항목을 수정했습니다.

### 수정 내역

#### 1. reviews 쿼리 에러 묵살 수정

- `error`를 destructure하지 않아 DB 오류 시 조용히 평점 0으로 반환되던 문제 수정
- `reviewError` 수집 후 `throw reviewError` 처리로 호출부에서 에러를 인지할 수 있도록 변경

#### 2. isLiked / isBookmarked 하드코딩 제거

- 항상 `false`로 하드코딩됐던 값을 실 데이터 조회로 교체
- `fetchRecipeDetail(id, userId?)` 시그니처 추가
- `userId`가 전달될 경우 `likes`·`bookmarks` 테이블을 `Promise.all`로 병렬 조회
- 기존 호출부(`page.tsx`, `route.ts`)는 `userId`가 옵셔널이라 수정 없이 호환

#### 3. reviewRows 다중 순회 개선

- `reduce`로 `ratingSum` + 별도 `for`로 `ratingBreakdown`을 두 번 순회하던 것을 단일 `for` 루프로 통합
- 순회 횟수를 2회 → 1회로 감소

#### 4. data 이중 타입 단언 제거

- `data as unknown as RawRecipe` → `data as RawRecipe`
- `supabaseAdmin`이 제네릭 없는 `SupabaseClient`라 `data`가 이미 `any`이므로 `as unknown` 중간 단계 불필요
