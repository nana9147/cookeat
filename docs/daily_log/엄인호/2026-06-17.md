## 세부 작업

### 1. 재료 쇼핑 목록 API 구현 (`app/api/products/route.ts`)

- **원인:** 기존 mock 데이터 기반 클라이언트 필터링으로는 실제 DB 데이터 표시 불가
- 카테고리 → `ingredient_id` 조회, 판매자 → `seller_id` 조회, 전체 판매자 목록을 `Promise.all`로 병렬 실행
- `별점순` 정렬은 Supabase 집계 미지원으로 최대 500행 메모리 정렬 처리
- `신상품순·낮은가격순·높은가격순·추천순(판매량)`은 DB ORDER + RANGE 페이지네이션

### 2. 재료 쇼핑 상세 API 구현 (`app/api/products/[productId]/route.ts`, `lib/products.ts`)

- `getProductDetail` 서버 유틸 함수 (`import 'server-only'` 보호)
- `product_images`(갤러리 이미지)·`reviews`(평점 집계)를 `Promise.all`로 병렬 조회
- `calcRating` 유틸 (`lib/utils.ts`) — 리뷰 없으면 0 반환, 소수 첫째 자리 반올림

### 3. `useShoppingFilter` 훅 서버 fetch 전환

- **원인:** mock `useMemo` 방식에서 API 호출로 전환 필요
- `useEffect` + `fetch /api/products` 방식으로 재작성
- `cancelled` 플래그 + `clearTimeout` — 요청 경쟁 조건 방지
- `prevPriceRef` 로 가격 필드 실제 변경 여부 감지 — 가격만 500ms 디바운스, 나머지 즉시 요청
- `shoppingFilter.ts`의 mock 기반 `filterAndSort`·`allSellers` 완전 제거

### 4. 상품 상세 페이지 API 연동 (`shopping/[id]/page.tsx`)

- `mockProducts` 참조 → `getProductDetail(productId)` 호출로 교체
- `id` 정수 검증 실패 시 `notFound()` 즉시 반환
- `details` 배열: `volume` 제거, `origin`·`brand` 실제 DB 필드로 교체

### 5. API 응답 타입 추가 (`types/ingredient.ts`)

- `ProductListItem` — 목록 API 응답 단일 상품
- `ProductDetail` — 상세 API 응답
- `ProductsResponse` — 목록 API 전체 응답 (products·sellers·pagination)

### 6. 컴포넌트 80줄 이하 분리

- `FilterSheetHeader` — 핸들·헤더(필터 카운트 배지·닫기 버튼) 분리
- `FilterSheetActions` — 하단 초기화·적용하기 버튼 분리
- `FilterSheet` 113줄 → 59줄
- `ShoppingProductArea` — 에러·스켈레톤·`ShoppingGrid`·`Pagination` 분리
- `sharedFilterProps` 객체 — `ShoppingFilter`·`FilterSheet` 공통 10개 props 단일화
- `ShoppingClient` 120줄 → 62줄
